# 添加语言

KolleK 目前提供七种语言：英语、法语、西班牙语、德语、巴西葡萄牙语、简体中文和日语。每个用户都可以在个人资料中选择自己的语言，甚至可以直接在登录页面切换。本页说明翻译机制在底层是如何运作的，以及运维者或贡献者应如何添加一种新语言，或补全一种尚未翻译完整的语言。

如果你只是想切换自己看到的界面语言，完全不需要了解这些内容。参见 @doc(profile.changeLanguage)。

## 翻译是如何存储的

每种语言对应 `lang/` 目录下的一个 JSON 文件，以语言代码命名，例如 `lang/fr_FR.json`。每个文件都把原始的英文字符串映射到对应的译文。应用提供的语言列表在应用配置中以“受支持语言”的形式定义。

## 搭建或刷新一种语言

`beaver:localize` 命令会扫描整个应用中所有可翻译的字符串，并将它们同步到对应语言的文件中：

```
php artisan beaver:localize fr_FR
```

自上次运行以来新增的字符串会被加入，不再存在的字符串会被移除。在英文文件中，每个字符串的译文就是它自己，因此英语在定义上永远是完整的。在其他每种语言中，新增的字符串会以空值出现，等待译者填写。

添加一种全新的语言遵循同样的流程：在受支持语言的配置中注册该语言，用新的语言代码运行该命令生成对应文件，然后翻译其中的空白条目。

:::note
空的译文会回退显示英文，而不会破坏界面，因此一种尚未翻译完整的语言在翻译工作继续进行的同时依然可以正常使用。
:::

## 目前还没有翻译的部分

登录后的应用界面是完全可翻译的。公开的营销站点和自动生成的 API 参考文档目前还没有翻译，无论访问者选择哪种语言，都始终以英文呈现。参见 @doc(troubleshooting.featureStatus)。
