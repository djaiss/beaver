# 实例管理面板

位于 `/instance-admin` 的实例管理面板，是 @doc(instanceAdmin.grantAccess, "实例管理员") 查看服务器上所有账户的地方：有多少个账户、每个账户里有谁，以及少数几个只有运维者才应该拥有的破坏性操作。本页说明这个面板能做什么，同样重要的是，它刻意不能做什么。

如果你运行的是只有一个账户的个人实例，可能永远都用不到这个面板。它真正发挥价值的场景，是拥有多个账户的共享实例，比如俱乐部或家庭服务器。

:::note
只有带有实例管理员标志的用户才能看到这个面板。其他任何人访问 `/instance-admin` 得到的都是未找到页面，而不是访问被拒绝页面，因此这个面板从不会透露自己的存在。
:::

## 概览页面

打开面板首先看到的是整个实例的概览：

- 整个服务器上**账户**、**用户**、**收藏**和**藏品**的数量统计。
- **本月新建账户数**和**本月活跃用户数**，让你了解实例是在增长还是趋于平静。
- 过去十二个月**每月注册数**的图表。

这些数字统计的是整个实例的情况，不会透露任何人具体的编目内容。

## 浏览账户

**账户**区域列出实例上的所有账户，每页 25 个，并显示每个账户的成员数量和收藏数量。

你可以**通过成员的邮箱地址**搜索账户，并按角色筛选。无法按账户名称或个人姓名搜索，因为姓名在数据库中是加密存储的，无法直接匹配。邮箱是唯一可靠的检索方式。

打开一个账户，可以看到其成员列表（先显示所有者，再显示编辑者，最后是查看者），以及该账户的收藏数量、藏品数量，还有最近的十五条活动日志。

## 破坏性操作

面板中有三项操作会更改或删除数据，而且都无法撤销：

- **删除账户**，会移除该账户以及其中的所有收藏、藏品、副本、成员和全部历史记录。
- **删除用户**，会将这个人从其账户中移除。
- **切换其他用户的管理员标志**，为其他人授予或撤销实例管理权限。

:::warning
在这个面板中删除账户或用户是立即且永久生效的。不会经过回收站，也无法恢复。确认操作之前，请务必再三核实你选中的账户或用户是否正确。
:::

有两项保护机制保护着实例本身：管理员无法撤销自己的标志，也无法在面板中删除自己的用户。无论这个面板如何被使用，实例始终会保留至少一位可用的管理员。

## 这个面板不是什么

这个面板在设计上仅限于 Web 端。JSON API 是限定在单个账户范围内的，没有实例级功能的容身之处，因此这些能力都不会以 API 接口的形式存在。

面板中可见的**支持**和**评价**区域目前只是占位内容，尚未实际构建。参见 @doc(troubleshooting.featureStatus)。
