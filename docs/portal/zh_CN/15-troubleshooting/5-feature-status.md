# 功能状态与路线图

KolleK 正在持续开发中，有一些功能在完全做好之前就已经可见了。这一页是唯一如实列出当前已完整可用和仍在开发中功能的清单，这样其他页面就不用含糊其辞了。产品每有进展，这一页也会同步更新。

## 现在已经可用

本文档站中记录的其他所有内容都按描述正常运作，包括：

- 收藏、藏品、副本、照片、标签、分类、套装，以及系列。
- 带自定义字段的收藏类型，包括以 JSON 格式导入导出类型定义。
- 完整的副本历史：交易、估值、保险、借还、保养、来源、位置历史，以及文档，全部通过统一的时间线呈现。
- 支持所有者、编辑者、查看者角色的协作功能，以及邮件邀请。
- 两步验证、魔法链接、API 密钥，以及安全提醒邮件。
- 完整的 JSON API，及其在 `/docs/api` 自动生成的参考文档。
- 基于 Docker 的自行托管、静态数据加密、带恢复功能的回收站，以及按收藏划分的统计信息。

## 尚未完成

### 全局搜索

仪表盘上的搜索框目前只是一个占位符，还搜索不了任何内容。目前可用的功能有：筛选你当前打开的收藏中的藏品（参见@doc(collections.chooseView)），以及搜索@doc(photos.library, "照片库")。

### 收藏的可见性和共享

每个收藏都带有一个可见性设置（私有、共享，或公开），这个设置会被保存下来，但目前还没有真正生效。账户中的每个成员仍然可以浏览账户里的每一个收藏，而且目前也没有公开链接，所以一个被标记为公开的收藏，实际上还是完全无法从账户外部访问的。现在设置可见性是在记录你的意图，等共享功能上线后就会生效。参见@doc(sharing.overview)。

### Webhook 投递

你可以注册 webhook 端点，每个端点都会获得一个签名密钥，但目前还没有任何应用内事件会触发 webhook。签名和投递机制已经就绪，只是还在等事件被接入。如果愿意，你现在就可以先设置好，等这部分功能逐步完善后，投递就会开始运作。参见@doc(webhooks.overview)。

### 藏品和收藏的导入导出

目前只支持收藏类型定义的导入导出，还没有单个藏品或整个收藏级别的导入导出功能。如果想导出全部数据，自行托管者可以使用完整的实例备份，参见@doc(dataSafety.backupCollectionData)。

### 实例管理：支持和评价

在实例管理面板中，"支持"和"评价"区域目前只是占位符，并且会明确说明这一点。面板的其余部分都可以正常使用，参见@doc(instanceAdmin.panel)。

## 如何理解这一页

这里的内容都不是带时间承诺的保证。"尚未完成"意味着相关基础工作可能已经存在，但在它被移入上面的列表之前，你不应该按这个功能已经可用来做规划。如果有疑问，请以这一页为准，而不是任何看起来暗示相反情况的内容。

如果这一页没能回答你的问题，很可能可以在@doc(troubleshooting.faq, "常见问题")中找到答案。
