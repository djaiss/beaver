# 副本历史详解

这一页解释的是 KolleK 的概念核心：一个副本展示的是它当前的状态，而它经历过的一切都保存在各自独立的、带日期的记录里。理解了这一点，整个"追踪"部分就都变成了一系列显而易见的操作。

## 当前状态与历史记录

看看 Priya 的一块手表。这个副本一眼就能告诉你它当前的状态：它的@doc(conditions.overview, "品相")是二手，它当前的@doc(locations.overview, "位置")是展示柜，它的估计价值就是最近一次鉴定给出的数字。

这些都不是直接输入的、会覆盖之前数值的简单事实，每一项都是背后某条记录露出的一角：

- 估计价值来自她**最近一次的估值**。
- 她付的价格和取得日期，来自她**最早的一笔取得性质的交易**。
- 当前位置是**位置历史中处于打开状态的那一条记录**。

副本本身是一份摘要，记录才是真相。

## 记录的类型

一个副本可以挂载七种带日期的记录，每一种都有自己的用途和对应的操作说明页面：

- **交易**记录金钱和所有权的变化：你付了多少钱、卖了多少钱、手续费、运费。参见@doc(copies.recordPaymentsAndValue)。
- **估值**记录副本在某个时间点值多少钱，以及是谁给出的这个判断。和交易在同一个页面，因为这两者很容易混淆。
- **保险记录**记录承保信息：承保方、保险价值、保单期限。参见@doc(copies.insure)。
- **借还**在副本离开你手中或从别人那里来到你手中时追踪保管权。参见@doc(loans.lendAndBorrow)。
- **保养记录**记录清洁、维修和保护性处理工作。参见@doc(copies.recordMaintenance)。
- **来源事件**构建所有权和真伪的故事。参见@doc(copies.traceProvenance)。
- **位置历史**记住这个副本存放过的每一个地方。参见@doc(copies.move, "移动副本")。

你还可以把@doc(copies.attachDocuments, "文档")（收据、鉴定报告、证书）附加到副本本身，或者附加到任意一条具体记录上，并在@doc(copyHistory.readTimeline, "副本时间线")中把所有内容合并到一起阅读。

## 保持一致性的两条规则

**金钱只存在于交易中。** 购买价格是一笔交易，出售也是一笔交易。估值和来源事件描述的是价值和故事，永远不涉及具体的付款。

**历史只能追加，不能覆盖。** 重新估值会在旧估值旁边写入一条新的估值记录，续保会写入一条新的保险记录。任何东西都不会覆盖过去，这正是时间线在多年之后依然能讲述完整故事的原因。

:::note
如果你发现自己正在编辑一条旧记录，好让它反映一些新情况，请停下来，改为添加一条新记录。编辑是用来纠正错误的，不是用来更新现实的。
:::

## 是不是所有这些都要用到

不需要。Emma 编目大多数漫画时，只用到副本、品相和位置这几项。历史记录真正体现价值的，是那些真正重要的藏品：贵重的、投了保的、借出去的，以及继承来的。每个副本该用多少历史记录，取决于它本身值得投入多少。
