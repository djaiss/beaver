# 核心概念

KolleK 有一套独特的数据模型，值得花二十分钟了解一下。只要弄懂这一节里的几个概念，后面所有的操作说明都会变得一目了然。建议先通读一遍这些页面，之后遇到术语不明白时再回来查阅。

请按以下顺序阅读：

1. @doc(kollek.howOrganized)：一页讲清整体的思维模型。
2. @doc(accounts.usersAndRoles)：工作空间、其中的成员，以及每个人能做什么。
3. @doc(collections.overview)：用来存放你要编目的所有内容的命名分组。
4. @doc(items.itemsVsCopies)：KolleK 中最重要的一个概念。
5. @doc(collectionTypes.overview)：如何决定每种藏品要记录哪些信息。
6. @doc(organizing.categoriesSetsAndSeries)：三种分组工具，各自回答不同的问题。
7. @doc(tags.overview)：跨藏品打标签和筛选的最轻量方式。
8. @doc(locations.overview)：如何对实体存放位置建模。
9. @doc(conditions.overview)：如何评定副本的品相。
10. @doc(copyHistory.concept)：价值、流转和来源都记录在哪里。
11. @doc(sharing.overview)：谁能看到某个收藏，以及什么时候能看到。
12. @doc(dataSafety.howProtected)：加密、审计日志，以及它们的局限性。

如果你只有五分钟，请阅读 @doc(items.itemsVsCopies)。这是后续文档始终依赖的唯一一页。
