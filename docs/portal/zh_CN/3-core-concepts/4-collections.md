# 收藏

收藏是一个由你命名的顶层分组，也是你开始编目时创建的第一样东西。这一页说明收藏包含哪些信息，以及创建收藏时需要做出哪些选择，让你可以放心地建立一个收藏。

## 收藏是什么

一个收藏容纳一组相关的@doc(items.itemsVsCopies, "藏品")。"相关"的含义由你决定：Emma 有一个"我的漫画"收藏，Noah 有一个"黑胶唱片"收藏，Priya 则有一个"腕表"收藏，还有一个叫"继承的藏品"的收藏。一个账户可以拥有任意多个收藏。

大多数收藏者会给每个爱好建一个收藏。如果想再细分（比如按出版商分），通常更适合在同一个收藏内部用@doc(organizing.categoriesSetsAndSeries, "分类")来实现，而不是拆成多个收藏。

## 收藏包含什么

创建或编辑一个收藏时，你可以选择：

- **名称**，例如"酒窖"。
- **描述**，可选，用来提供背景信息。
- **表情符号**，从一组固定的表情中选择（📚 代表图书，💿 代表音乐，🍷 代表葡萄酒，等等），会显示在列表中名称旁边。
- **货币**：每个收藏都有自己的货币，从支持的货币列表中选择（USD、EUR、GBP、JPY 等）。它可以和账户的默认货币不同，这在你用欧元买黑胶唱片、用美元买葡萄酒时会很方便。这个收藏中副本记录的金额都会使用它。
- **可见性**：私有、账户内共享，或通过链接公开。可见性值得单独说明，参见@doc(sharing.overview)。
- **一个或多个@doc(collectionTypes.overview, "收藏类型")**：这决定了这个收藏中的藏品可以使用哪些自定义字段。在一个漫画收藏上启用漫画类型，每个藏品就都能记录期号和出版商了。

只有名称是必填的，其他一切都可以之后再调整。

## 收藏内部有什么

一个收藏拥有它自己的藏品、@doc(organizing.categoriesSetsAndSeries, "分类")，以及@doc(organizing.categoriesSetsAndSeries, "套装")。它还会为每个成员分别记住各自偏好的查看方式（网格、列表，或表格），所以 Emma 的表格视图不会影响到 Sam 的网格视图。

## 接下来可以做什么

- 在@doc(items.itemsVsCopies)中了解收藏内部装的是什么。
- 在@doc(collections.manage)中实际创建一个收藏。
- 在@doc(collections.share)中决定谁可以看到它。
