# 分类、套装和系列

KolleK 提供了三种给藏品分组的方式，它们看起来很相似，直到你弄清楚每一种各自回答的是什么问题。这一页把它们理清楚，让你每次都能选对工具。

## 三个问题

- **这个藏品该归到哪里？** 这是**分类**要回答的问题。
- **我正在追求集齐哪份有限的清单？** 这是**套装**要回答的问题。
- **这个东西在我收藏的所有内容里，属于哪个系列？** 这是**系列**要回答的问题。

## 分类：在一个收藏内部归档

分类把藏品归档在一个@doc(collections.overview, "收藏")内部，分类可以嵌套。Emma 的"我的漫画"收藏里有一个 Marvel 分类，里面又嵌套了蜘蛛侠和 X 战警两个子分类。浏览一个分类时，只会显示它自己的藏品，以及各自的数量和统计信息。

一个分类属于它所在的收藏，无法跨越到收藏之外。删除一个分类不会删除里面的藏品，它们只会变成未分类状态。

## 套装：一份有限的待集清单

套装是一份有终点的清单。Emma 想要集齐某个系列的前十期，于是她创建了一个**目标数量**为十的套装。当她往套装里添加藏品时，KolleK 会显示已拥有数量对比目标数量，完成情况也会汇总进她的@doc(insights.collectionStatistics, "收藏统计")中。

和分类一样，一个套装也属于一个收藏。只有目标数量大于零的套装才会计入完成度统计，没有设置目标数量的套装只是一个命名分组而已。

## 系列：跨收藏的系列作品

系列把来自同一个 IP 系列的所有内容串联在一起，不管它们分散在哪些收藏里。哈利波特跨越了 Emma 的图书、电影和乐高积木这三个不同的收藏。系列会横跨所有这些收藏，它的页面会按每个藏品所在的收藏分组显示所有关联藏品，并展示它一共涉及了多少个收藏。

系列不追踪完成度，它回答的是"给我看看所有和哈利波特有关的东西"，而不是"我集齐了吗"。

## 三者对比

| | 分类 | 套装 | 系列 |
|---|---|---|---|
| 范围 | 单个收藏 | 单个收藏 | 整个账户 |
| 可嵌套 | 可以 | 不可以 | 不可以 |
| 追踪完成度 | 不追踪 | 追踪，带目标数量 | 不追踪 |
| 回答的问题 | 归到哪里 | 属于哪份清单 | 属于哪个系列 |

一个藏品可以同时使用这三种方式：归档在 Marvel 分类下，属于"前十期"套装，同时又关联到蜘蛛侠系列。

如果你想要比这些更轻量、完全没有结构的跨领域标签，那就是@doc(tags.overview, "标签")的用途了。

## 接下来可以做什么

- 在@doc(categories.organizeItems)中搭建一棵归档树。
- 在@doc(sets.trackCompletion)中追踪集齐一整套的进度。
- 在@doc(series.groupFranchise)中把一个系列串联起来。
