# 标签

标签是给藏品打标记最轻量的方式：没有层级，没有结构，只是一些你附加上去、之后可以用来筛选的词语。这一页说明标签擅长做什么，以及它和其他分组工具的区别。

## 标签是什么

标签是一个简短的、自由格式的标记，比如"已签名"、"首期"，或"待投保"。标签在整个账户的所有收藏之间共享：创建一次"已签名"标签，它就同时可以用在 Emma 的漫画上，也可以用在 Noah 的黑胶唱片上。一个藏品可以携带任意多个标签。

标签最擅长表达那些贯穿你整个目录的特质。"已签名"既适用于漫画，也适用于图书和唱片。@doc(organizing.categoriesSetsAndSeries, "分类")永远做不到这一点，因为分类只存在于单个收藏内部。

## 标签与其他分组方式的区别

- **分类**把一个藏品归档在一个收藏内部的一个位置，并且可以嵌套。标签不设任何层级，可以用在任何地方。
- **套装**追踪一份有限清单的完成度，标签不计数任何东西。
- **系列**把一个 IP 系列串联在一起，标签除了它本身这个词以外，不携带任何额外含义。

一个实用的判断规则：如果这个标记回答的是"这个东西该归到哪里"，就用分类；如果它标记的是一种特质或状态（"已签名"、"重复"、"待出售"），就用标签。

## 标签从哪里来

任何拥有编辑者或所有者权限的人，都可以在编辑藏品时随手创建一个新标签，或者从已有列表中选择。所有者和编辑者也可以在账户设置里维护这份共享列表，详见@doc(tags.manageAccount)。

## 删除一个标签

删除一个标签，会把这个标记从所有携带它的藏品上移除，藏品本身不受影响。

:::note
标签删除是账户范围的操作。删除"已签名"，它会同时从账户内所有收藏的所有藏品上消失。
:::

## 接下来可以做什么

- 在@doc(items.tagAndFind)中应用标签并按标签筛选。
- 在@doc(tags.manageAccount)中维护共享标签列表的整洁。
- 在@doc(organizing.categoriesSetsAndSeries)中和结构化的分组方式做对比。
