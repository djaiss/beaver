# 位置

位置回答的是每个不断增长的收藏迟早都会遇到的问题："我把它放哪儿了？" 这一页说明 KolleK 是如何为实物存放建模的。

## 位置是什么

位置是一个@doc(items.itemsVsCopies, "副本")实际存放的地方：一个房间、一个架子、一个盒子、一个保险柜。每个位置都可以配一个表情符号，这样在列表中一眼就能认出来。

位置可以按你需要的深度任意嵌套，从而映射出你真实的存放空间。Noah 是这样建模的：客厅，客厅里再有 A 号架子，A 号架子里再有 3 号箱。当他想知道某张唱片放在哪儿时，答案会和他的地图一样精确。

位置是账户级别的。定义一次"展示柜"，每个收藏都可以把副本存放在那里，这也符合现实：同一个架子上可以并排放着漫画和硬币。

## 位置绑定的是副本，而不是藏品

藏品是一个概念，实际占据空间的是副本。Emma 同一本漫画的两个副本存放在不同的地方：一本放在 1 号长盒里，一本裱框挂在墙上。每个副本都指向自己当前的位置。

一个全新的账户会自带几个起始位置（客厅、储藏室、展示柜、车库、办公室）。你可以重命名它们、在它们下面嵌套子位置，或者用自己的位置替换它们。

## 移动会被记住

当你移动一个副本时，KolleK 不会只是简单地覆盖旧的位置，而是会记录这次移动，这样副本就保留了一条记录着它去过哪里、何时去的完整轨迹。当前位置只是这条轨迹里最新的一条记录而已。这属于@doc(copyHistory.concept, "副本历史")的一部分，具体操作方法在@doc(copies.move)中。

## 接下来可以做什么

- 在@doc(locations.setup)中搭建你的存放位置地图。
- 在@doc(copies.move)中正确地移动物品。
- 在@doc(copies.track)中查看位置字段在副本表单上的位置。
