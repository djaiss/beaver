# 可见性和共享

每个收藏都带有一个可见性设置，记录它是给谁看的。这一页说明三个级别分别是什么，也同样重要的是，说明这些设置目前实际生效的程度是多少，这样无论朝哪个方向，你都不会感到意外。

## 三个级别

可见性是分别设置在每一个@doc(collections.overview, "收藏")上的，所以同一个账户可以混用这三种：

- **私有**：只给你自己看，适合任何私人内容，或者还在建设中的目录。
- **共享**：给你@doc(accounts.usersAndRoles, "账户")里的每一个成员看，无论是所有者、编辑者，还是查看者。
- **公开**：给你把链接发给的任何人看，无需登录，只读。

Priya 把"继承的藏品"设为私有，把"腕表"设为和家人共享，把她想展示的俱乐部目录设为公开。

## 目前实际生效的程度

可见性目前只是一个被记录下来的意图，还不是一道真正生效的门槛。由此产生两个方向的后果：

- **在账户内部，每个成员都能浏览每一个收藏**，不管它的可见性设置是什么。把一个收藏设为私有，目前还不会把它从账户其他成员那里隐藏起来。
- **从账户外部则完全无法访问。** 目前还没有公开的收藏页面，所以即使一个收藏被设为公开，也没有链接可以发给别人，账户之外的任何人都看不到它。从这个角度看，你的数据其实比这个设置字面上暗示的还要私密。

现在设置可见性，是在记录你的意图，这样等共享功能真正上线时，你的目录会立刻按你的预期运作。这项功能的状态会持续维护在@doc(troubleshooting.featureStatus)中。

:::note
如果你和其他人共用同一个账户，目前请假设他们能看到账户里的每一个收藏，包括标记为私有的那些。真正私人的内容，请放在你自己独立的账户里。
:::

## 公开会意味着什么

当公开链接功能上线后，一个公开的收藏将可以被任何拿到链接的人在任何地方查看，无需登录。这是一个值得慎重决定的选择，所以这个设置是挂在每一个收藏上的，而不是一个统一的总开关。

## 可见性和角色是配合使用的

可见性关乎的是"能不能看"，@doc(accounts.usersAndRoles, "角色")关乎的是"能不能改"。你账户里的一个查看者，不管收藏的可见性是什么，都只能浏览，不能编辑任何内容。一个通过公开链接访问的访客根本不是成员，所以不会套用任何角色，也就无法编辑任何东西。

## 接下来可以做什么

- 在@doc(collections.share)中一步步设置收藏的可见性。
- 在@doc(collaboration.manageMembersAndRoles)中查看账户里都有谁。
- 想知道你的数据是如何存储的？参见@doc(dataSafety.howProtected)。
