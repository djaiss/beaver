# 登录与安全提醒邮件

有时候，KolleK 会在你没有主动请求的情况下给你发邮件。这些提醒的目的，是确保当你的用户身上发生了什么事情时，你能第一时间从 KolleK 那里得知，而不是通过其他渠道。本页列出每一种提醒、它们各自意味着什么，以及如果收到意外的提醒时该怎么做。

## 登录失败

**什么时候会收到：** 有人在登录页面用你的邮箱输入了错误的密码。

**如果是你自己**，只是不小心打错了密码，可以忽略它。

**如果不是你**，说明有人在尝试你的账户。偶尔一次失败通常只是噪音，但反复收到这个提醒，说明你的邮箱正在被针对性尝试。请确保你的密码是 KolleK 专用的，并开启 @doc(security.twoFactorAuth, "两步验证")，这样即使密码被猜中也不够用。

## 新设备登录

**什么时候会收到：** 一次成功的登录发生了，邮件中会说明它来自哪个设备。

**如果是你自己**，是在新的浏览器、手机或电脑上登录，可以忽略它。

**如果不是你**，说明有人拿到了你的密码，请立刻 @doc(auth.resetPassword, "修改密码")，并检查账户里有没有异常内容。

## IP 地址变化

**什么时候会收到：** 你这次登录使用的网络地址和上一次不同。

这在你出差、切换网络，或者你的运营商更换了地址时都很正常。只有当它和一次你不认识的登录同时出现时，才值得留意。

## API 密钥创建、API 密钥删除

**什么时候会收到：** 你的用户名下有一个 @doc(apiKeys.manage, "API 密钥") 被创建或被撤销了。

**如果是你自己**在管理密钥，可以忽略它。

**如果不是你**，请认真对待。一个意外出现的密钥，说明有人拥有足够的访问权限来创建它。请撤销这个密钥、修改密码，并检查你剩下的密钥和它们各自最后一次使用的时间。

:::note
你通过 API 登录时在后台创建的登录令牌，不会触发密钥创建邮件。只有你手动创建的密钥才会触发，这样提醒才始终具有实际意义。
:::

## 你自己请求过的邮件

还有两种邮件只会在有人主动请求时才会发送，所以它们本身不算提醒：@doc(auth.magicLinks, "魔法链接") 邮件和密码重置邮件。如果你收到了一封自己没有请求过的这类邮件，说明有人在对应的表单里输入了你的地址。这两种邮件都无法在没有访问你邮箱的情况下被使用，但反复收到未请求的邮件，同样是你的地址正在被人试探的信号。

## 如果确实觉得不对劲

1. @doc(auth.resetPassword, "修改密码")。
2. 如果还没开启，打开 @doc(security.twoFactorAuth, "两步验证")。
3. 检查你的 @doc(apiKeys.manage, "API 密钥")，撤销任何你不认识的密钥。
4. 查看 @doc(activity.logAndSentEmails, "你的个人活动日志")，确认有没有你没做过的操作。
