# 教程：为特定爱好配置你的账户

添加一件藏品很容易。但要顺利添加两百件，前提是账户已经提前配置好。在本教程中，你会在批量录入数据之前，针对特定爱好定制 KolleK：调整收藏类型和它的自定义字段，建立一套还原你真实空间的位置结构，并预先创建一套标签词汇表，这样之后添加的每一件藏品都能又快又一致。

我们将跟随即将编目大约三百张黑胶唱片的 Noah 一起操作。同样的方法适用于任何爱好，操作时把内容替换成你自己的即可。

预计这个过程需要大约半小时，但能为你之后节省许多小时。

## 开始之前

- 完成 @doc(tutorials.catalogueFirstCollection, "端到端编目你的第一个收藏")，或者至少完成 @doc(gettingStarted.quickStart, "快速上手")，熟悉核心操作流程。
- 了解 @doc(collectionTypes.overview, "收藏类型与自定义字段")、@doc(locations.overview, "位置") 和 @doc(tags.overview, "标签") 背后的概念。如果还不熟悉，先快速浏览一下这些页面。
- 花点时间想清楚你到底想为每件藏品记录什么。花十分钟用笔记本理清思路，胜过在录入五十条记录之后再返工调整字段。

## 第一步：调整收藏类型

Noah 从账户自带的**黑胶唱片**预置类型开始。它已经包含了我的评分，一个**发行信息**分组（艺人、专辑、发行年份），以及一个**压制细节**分组（压制版本、转速、彩胶）。

这已经很接近他想要的效果，但他买了很多日本压制版，也很在意封套的品相，于是他调整了这个类型。

::::steps
:::step title="打开这个类型"
进入收藏类型设置，选择**黑胶唱片**。编辑器会随手操作自动保存，不需要寻找保存按钮。

::screenshot{label="显示黑胶唱片字段的收藏类型编辑器"}
:::

:::step title="添加你实际会用到的字段"
Noah 在压制细节分组中添加了一个文本字段**压制国家**，并添加了一个选择型字段**封套品相**，选项按他自己的评级标准设置。可用的字段类型有文本、数字、日期、是非、选择和评分（最多五星）。
:::

:::step title="分组并排列字段顺序"
如果一组字段属于同一个主题，就为它们新建一个分组，并通过拖拽把字段排列成你想要的顺序。分组的唯一作用，就是让长表单保持易读。
:::
::::

为什么这一步很重要：现在定义的自定义字段，会出现在所有使用这个类型的收藏中的每一张藏品表单上。提前定好这些字段，意味着你会得到三百条一致的记录，而不是三百条随手拼凑出来的记录。

:::note
设计字段时要考虑你以后会问的问题。“哪些唱片是彩胶”这个问题，只有在“彩胶”是一个字段时才能被回答。埋在描述文字里的细节是无法被筛选出来的。
:::

## 第二步：建立你的位置结构

Noah 把唱片存放在两个地方：一间有三层架子的听音室，以及储藏室里的几个箱子。他把这个结构原样还原了出来，因为 KolleK 中的位置，只有在对应一个你能实际走到的真实地方时才有意义。

::::steps
:::step title="创建顶层位置"
在 @doc(locations.setup, "位置设置") 中，创建**听音室** 🛋️ 和**储藏室** 📦，这两个是房间。
:::

:::step title="嵌套真实的细分位置"
在听音室下面，创建**A 架**、**B 架**和**C 架**。在储藏室下面，创建**1 号箱**和**2 号箱**。位置可以按需要嵌套任意深度，所以房间里放着箱子、箱子里再放着盒子，都是没问题的。
:::
::::

为什么这一步很重要：每个副本都指向一个位置，之后的移动都会被记录为 @doc(copies.move, "位置历史")。现在建立一套完善的位置结构，意味着“那张唱片放在哪儿”这个问题永远有一个准确的答案。

## 第三步：预先建立标签词汇表

标签能够跨越收藏和层级结构，因此非常适合用来标记那些没有其他归属的标签。Noah 在 @doc(tags.manageAccount, "标签设置") 中创建了他的初始标签集：**已签名**、**首版压制**、**日本压制**、**待出售**和**需要清洁**。

有两个习惯能让标签一直保持有用：

- 数量要少，且可复用。一个只用过一次的标签，本该是一个字段或一条备注。
- 在其他人加入之前，先统一好拼写。“已签名”和“亲笔签名”作为两个不同的标签，之后会一直困扰你。

编辑藏品时，你随时可以现场创建一个新标签，所以这份初始清单只需要覆盖你已经确定想要的标签即可。

## 第四步：导入类型，而不是从头搭建

这里有一个值得了解的捷径。收藏类型可以 @doc(collectionTypes.importExport, "以 JSON 格式导出和导入")。如果朋友已经搭建好了一个出色的黑胶类型，他可以导出它，你只需粘贴这段 JSON 就能导入，一步到位地带入名称、颜色、分组、字段和选择项。

:::note
导入类型只会带入类型定义本身，不会导入任何藏品或藏品数据。目前还没有藏品导入或整个收藏导入的功能，具体现状记录在 @doc(troubleshooting.featureStatus, "功能状态页面") 中。
:::

Noah 导入了俱乐部朋友分享的“45 转单曲”类型，它出现在他自己的类型旁边，可以直接关联到某个收藏使用。

## 第五步：创建收藏，把一切连接起来

现在把这些准备工作整合到一起。

::::steps
:::step title="创建收藏"
Noah 创建了一个名为“Vinyl”的收藏，选择了 💿 表情符号，并写了一句简短的描述。
:::

:::step title="启用需要的类型"
他同时启用了**黑胶唱片**类型和刚刚导入的**45 转单曲**类型。一个收藏可以使用多个类型，每件藏品选择适合自己的那一个。
:::

:::step title="设置货币"
他把这个收藏的货币设置成自己实际购买唱片时使用的货币。它可以和账户默认货币不同，这个收藏中副本的所有金额都会以这个货币显示。
:::
::::

## 成果如何

现在添加一件藏品，感受一下区别：表单会准确地询问该问的问题，位置下拉列表提供的是真实存在的架子，而你需要的标签也早已存在。从这里开始，批量录入数据会变成一种自然的节奏，而不是一连串需要临时决定的事情。

## 需要避免的常见错误

- **过度设计字段。** 十个你会认真填写的字段，胜过二十五个你会跳过的字段。字段可以之后再添加，但事后回填才是真正麻烦的部分。
- **位置和现实不符。** 如果现实中根本没有 B 架，那么“B 架”这个位置很快就会失去意义。
- **用标签做字段更擅长的事。** 等级、年份或评分应该放进自定义字段，成为真正的数值，而不是一个标签。
