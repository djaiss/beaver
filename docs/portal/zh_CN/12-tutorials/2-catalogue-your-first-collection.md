# 教程：端到端编目你的第一个收藏

在本教程中，你将从一个全新的账户开始，一步步建立起一个真实、内容完整的收藏。你会创建一个收藏，了解它记录哪些自定义字段，添加一件带封面照片的藏品，记录你实际拥有的实体副本，登记你为它支付的金额，添加第一次估值，最后查看由此生成的统计数据。

我们将跟随收藏漫画的 Emma 一起操作。她之前在 @doc(gettingStarted.quickStart, "五分钟快速上手") 中快速编目过一件藏品，这次她会认真完整地走一遍流程，最终她的目录会记录下这本漫画花了多少钱、值多少钱，以及存放在哪里。

预计这个过程需要二十到三十分钟。

## 开始之前

- 你需要一个能够登录的账户。如果还没有，请先 @doc(accounts.create, "创建你的账户")。
- 你应该了解藏品和副本之间的区别。如果不确定，请先阅读 @doc(items.itemsVsCopies, "藏品与副本")，本教程会反复用到这个概念。
- 准备一件你实际拥有、打算编目的东西，最好有一张照片，并大致记得当初的购买价格。

## 第一步：创建收藏

每一件藏品都存在于一个 @doc(collections.overview, "收藏") 之中，所以一切都从这里开始。

::::steps
:::step title="新建一个收藏"
在仪表盘中，选择**新建收藏**。

::screenshot{label="带有新建收藏按钮的仪表盘"}
:::

:::step title="命名并设置图标"
Emma 将她的收藏命名为“My Comics”，选择了 📚 表情符号，并写了一句简短的描述。表情符号和描述都是可选的，但能让收藏在之后更容易一眼认出。
:::

:::step title="选择收藏类型"
为这个收藏启用系统预置的**漫画**类型。收藏类型决定了这个收藏里的藏品可以记录哪些自定义字段。
:::

:::step title="暂时不用调整可见性和货币"
新建的收藏默认是**私密**的，只有你自己能看到，并且使用账户的默认货币。这两项之后都可以更改。保存这个收藏。
:::
::::

为什么这一步很重要：这张表单上的选择会影响后续的一切。类型决定了每件藏品需要填写哪些字段，货币则决定了这个收藏中副本的金额如何显示。

## 第二步：查看漫画类型会记录哪些内容

你的账户自带了十几种预置的 @doc(collectionTypes.overview, "收藏类型")。在添加藏品之前，先看看漫画类型会向你询问哪些内容，这样在填写藏品表单时就不会遇到意外。

打开收藏类型设置，选择**漫画**。你会看到：

- **我的评分**，一个五星评分字段。
- **出版信息**分组：期号（数字）、出版商（可选 Marvel、DC、Image、Dark Horse 或独立出版）、编剧、画师和封面日期。
- **品相与鉴定**分组：变体版和是否签名，都是是非选择题。

你不需要更改任何内容。如果想添加或调整字段顺序，可以参考 @doc(collectionTypes.setup, "类型设置指南")。就本教程而言，默认设置正好满足 Emma 的需要。

## 第三步：添加藏品，填写详情并上传照片

现在是最有成就感的部分。打开你新建的收藏。

::::steps
:::step title="创建藏品"
选择添加一件**新藏品**，并为它填写**名称**。Emma 输入了“Amazing Spider-Man #300”。
:::

:::step title="填写自定义字段"
因为这个收藏使用了漫画类型，表单会显示你刚才看到的那些字段。Emma 将**期号**设为 300，**出版商**选择 Marvel，**是否签名**选择否。填写你知道的信息，其余留空即可，空字段没有问题。

::screenshot{label="显示漫画自定义字段的藏品表单"}
:::

:::step title="上传封面照片"
为藏品添加一张**照片**。支持最大 10 MB 的 JPEG、PNG、WebP 和 GIF 文件。如果添加了多张，把最满意的一张标记为主照片，它会成为你在各个列表中一眼认出这件藏品的封面。
:::
::::

为什么这一步很重要：期号、出版商这类描述性信息属于藏品本身，因为它们对这本漫画在全世界的每一本副本都成立。到目前为止，你填写的任何信息都还没有涉及 Emma 手中那本实体副本，接下来就是它的部分。

## 第四步：记录你拥有的副本

没有 @doc(items.itemsVsCopies, "副本") 的藏品，只是百科全书里的一个词条。副本才是你实际拥有的实体物品。

::::steps
:::step title="为藏品添加副本"
在藏品页面上，添加一个**副本**。
:::

:::step title="评定品相并安排存放位置"
设置它的**品相**。Emma 从系统预置的列表中选择了**旧品**（每个账户都自带全新、几乎全新、旧品、磨损和损坏这几个选项）。然后设置它的**位置**。Emma 把它存放在**储藏室**，这是默认位置之一，不过你随时可以 @doc(locations.setup, "自定义自己的位置结构")。
:::

:::step title="确认状态"
将状态保持为**持有中**。其他状态（已订购、已外借、已售出等）是为那些当前不在你手边的副本准备的。保存。
:::
::::

:::note
如果你拥有同一本漫画的两本，不要再创建一个新藏品，而是为这个藏品添加第二个副本。每个副本都有各自独立的品相、位置、金额和历史记录。
:::

## 第五步：记录你的购买花费

这就是 KolleK 超越普通清单的地方。金额信息从不记录在藏品上或某条备注里，而是记录在副本的一笔**交易**中，这样即使记录越来越多，数据也依然精确。完整说明参见 @doc(copies.recordPaymentsAndValue)。

::::steps
:::step title="打开副本的历史记录"
打开藏品的**历史**标签页。它每次显示一个副本的记录，目前你只有一个副本。
:::

:::step title="添加一笔购买交易"
添加一笔类型为**购买**的**交易**。Emma 输入了她支付的金额、作为交易对象的店铺，以及购买日期。如果需要，还可以填写税费、手续费、运费和参考编号。

::screenshot{label="选中购买类型的新建交易表单"}
:::
::::

为什么这一步很重要：这笔购买交易正是副本**支付价格**和**获得日期**的来源。第七步中你会看到的统计数据，正是依据这个日期来绘制你收藏的增长曲线的。

## 第六步：添加第一次估值

你支付的金额和它现在值多少钱是两件不同的事，KolleK 刻意将它们分开记录。价值以**估值**的形式记录。

同样在副本的历史记录中，添加一次**估值**。Emma 选择了**自我估计**这个类型，输入她认为这本漫画现在能卖多少钱，并将置信度设为**中等**。等她某天请专业人士鉴定之后，她会添加一次新的估值，而不是修改这一次，旧的估计会作为历史记录保留下来。

:::note
购买价格永远是一笔交易，而不是估值。副本的估计价值始终来自它最近一次的估值，而支付价格则来自它最早的获得交易。
:::

## 第七步：查看你的成果

打开这个收藏，你应该能看到：

- 你的藏品带着封面照片，出现在网格视图中。
- 藏品数量为一件，总价值与你的估值一致。

现在打开这个收藏的**统计**页面。即使只有一件藏品，也已经有一些数据可以查看：预估总价值、按位置划分的价值，以及你购买它那个月对应的获得记录。@doc(insights.collectionStatistics, "统计指南") 解释了每一个数字的来源。

## 你完成了什么

你完整走完了 KolleK 的核心流程：一个带有类型的收藏，一件带有详情和照片的藏品，一个带有品相和位置的副本，一笔记录金额的交易，一次记录价值的估值。产品中的每一项功能，都建立在你刚刚创建的这些记录之上。

## 需要避免的常见错误

- **创建重复的藏品，而不是副本。** 同一本漫画的两本，应该是一件藏品带两个副本。
- **把购买价格填成估值。** 你支付的价格应该是一笔购买交易，估值记录的是它现在的价值。
- **把副本的信息填在藏品上。** 品相、位置和金额始终属于副本，因为第二个副本在这三方面都可能不同。
