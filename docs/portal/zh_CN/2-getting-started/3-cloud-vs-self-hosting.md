# 云托管还是自托管

在你投入时间使用 KolleK 之前，先了解一下你将如何运行它会很有帮助。本页将介绍你的可选方案，同样重要的是，说明你不需要为哪些东西付费。

## 没有订阅制

KolleK 没有套餐、没有分级，也没有需要付费才能解锁的功能。应用内完全没有计费系统。无论你以哪种方式运行它，得到的都是同一款完整的应用，不存在需要升级才能解锁的内容。

:::note
KolleK 内没有任何需要购买的东西。自托管是免费的，无论你以哪种方式运行，所有功能都已经包含在内。
:::

因此，下面要做的选择只关乎软件运行在哪里，而不关乎你能用它做什么。

## 方案一：自行托管

这是运行 KolleK 的主要推荐方式，并且完全免费。

你使用 Docker 在自己的服务器或电脑上运行 KolleK。你的目录、照片和文档都存放在你自己掌控的基础设施上。备份存放在哪里由你决定，而且随时都可以迁移全部数据。

如果你能自如地运行一个小型网络应用，或者愿意学习，自行托管会很适合你。安装过程简短，并且有完整文档说明。

如果这正是你的情况，安装指南会放在本文档的**自托管**部分。目前，实际的入门起点是项目的 Docker 指南 `docker/README.md`。

## 方案二：使用他人托管的实例

如果你不想自己运行任何东西，也可能有人愿意为你托管一个 KolleK 实例。这只是应用之外的一种便利安排，不会改变软件本身，不会解锁任何额外功能，也不会产生任何应用内费用。

如果你使用他人托管的实例，就可以完全跳过安装步骤，直接创建你的账户。

## 该如何选择

如果你希望完全掌控自己的数据，并且乐于运行一个小型服务，就选择自托管。如果你更希望由别人来维护服务器，就选择他人托管的实例。无论哪种方式，你使用的应用都完全相同，而且以后随时可以迁移数据，因为你可以导出并备份它。
