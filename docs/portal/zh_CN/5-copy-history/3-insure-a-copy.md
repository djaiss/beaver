# 为副本投保

当一个副本价值高到值得投保时，保单信息就应该和它所保障的东西放在一起，而不是塞在抽屉里。保险记录把承保方、保险价值和保单期限都保存在副本本身上，让你随时清楚它受到什么保障，以及保障到什么时候。

这是@doc(copyHistory.concept, "副本历史详解")中介绍过的记录类型之一。和其他记录一样，它也是历史：一个副本可以随时间拥有多条保险记录，续保时会写入一条新记录，而不是覆盖旧的。

## 什么时候使用它

当确实存在一份真实保单时，就在 KolleK 中为该副本记录保险，不论这个副本是有专门的保单承保、被单独列在一份更大的保单清单上，还是通过附加条款纳入了你的家庭财产保险。这条记录只是你的参考，并不是保单本身，它提醒你申报了什么、申报金额是多少，以及什么时候到期。

Priya 为她最贵重的一块表，也就是最近估值为 5500 元的 1968 年欧米茄超霸表，通过一家收藏品保险公司投了保。以下是她记录的过程。

## 添加一条保险记录

::::steps
:::step title="打开副本的历史记录"
打开藏品，进入它的**历史**标签页，选择该副本，打开**保险**部分。

::screenshot{label="展开了保险部分的历史标签页"}
:::

:::step title="记录保单基本信息"
添加一条保险记录，填写**承保方**、**保险价值**和**保单号**。
:::

:::step title="描述承保范围"
用自己的话填写**承保类型**（例如"单独列明的藏品"或"全险"）、**免赔额**，以及**起保日期**和**到期日期**。标记这个副本是否为**单独列明的藏品**，也就是它是否在保单上被单独列出。
:::

:::step title="添加保险公司的联系方式和状态"
记录保险公司的**联系方式**，这样理赔时就不用翻找旧邮件了，再设置**状态**并保存。
:::
::::

## 状态

一条保险记录始终处于以下四种状态之一：

- **生效中**：保单当前正在承保这个副本。
- **待生效**：已经安排好，但尚未生效。
- **已过期**：到期日期已过。
- **已取消**：在保险期限结束前就终止了。

已过期和已取消的记录会保留在历史中，但会以灰显的方式排在当前记录之后。这是有意为之：一份旧保单仍然是这个副本在过去这些年里价值和承保情况的有力证据。

:::note
当保单以新的价值续保时，请添加一条新的保险记录，而不是编辑旧的那条。这一系列记录展示了承保范围是如何随着副本价值的变化而变化的。
:::

## 把单据一并附上

保险记录一旦附上保单文件，用处会大得多。你可以直接在保险记录上@doc(copies.attachDocuments, "附加文档")，这样需要的时候，保单条款或保险证明就是一键可及的。

如果你是按专业鉴定的价值投的保，也建议把那次鉴定记录为一条类型为**保险价值**的估值。参见@doc(copies.recordPaymentsAndValue)。

## 接下来可以做什么

- 在@doc(copies.attachDocuments)中把保单条款附加到这条记录上。
- 在@doc(copies.recordPaymentsAndValue)中记录承保所依据的那次估值。
- 在@doc(copyHistory.readTimeline)中查看保险记录和其他记录一起呈现的样子。
