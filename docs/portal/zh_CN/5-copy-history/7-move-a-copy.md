# 移动副本并保留位置历史

当你在创建副本时选择了一个@doc(locations.overview, "位置")，你就告诉了 KolleK 它存放在哪里。移动是另一回事：它改变了副本现在的存放位置，*同时记住它以前放在哪里*。年复一年，这就积累成一条记录着这个副本曾被存放过的所有地方的轨迹。

## 位置历史是如何运作的

一个副本的位置历史是一条记录链。在任意时刻，都只有一条记录处于打开状态，那条打开的记录就是副本当前的位置。记录一次移动会关闭上一条记录，并打开一条新记录，标注上日期和原因。

所以"它现在在哪"和"它曾经在哪"其实是同一份数据，只是从两个不同的方向去读而已。

## 记录一次移动

Priya 在给她的 1968 年欧米茄超霸表重新估值之后，把它从书桌抽屉移到了保险柜。

::::steps
:::step title="打开副本的历史记录"
打开藏品，进入它的**历史**标签页，选择该副本，打开**位置**部分。

::screenshot{label="展开了位置部分的历史标签页"}
:::

:::step title="选择要移动到哪里"
记录一次移动，从你账户已有的位置中选择**新位置**。如果合适的位置还不存在，所有者或编辑者可以先添加它（参见@doc(locations.setup)）。
:::

:::step title="说明时间和原因"
设置移动的**日期**、**原因**（例如"估值提高，移入保险柜"），以及任何**备注**。保存。
:::
::::

现在这个副本会显示当前位置为保险柜，而抽屉那条记录则变成了历史，带有起始日期、结束日期，以及离开的原因。

## 修正错误

修正一条过去的记录是可行的。如果你日期打错了，或者选错了架子，请直接编辑那条记录，而不是再添加一次虚构的移动来弥补。历史应该反映真实发生过的事情。

:::note
移动是针对真实的物理位置变动。如果你只是在重新整理存放体系本身，比如重命名一个架子，或者把一个盒子挪到另一个位置下面，请改在@doc(locations.setup)中修改这个位置本身。副本会继续指向它，不需要记录任何移动。
:::

## 为什么要费心记录历史

对于普通的副本来说，你日常只会关心当前位置。位置历史真正体现价值的，是那些贵重的藏品：保险公司可能会问它一直存放在哪里，买家可能在意它在恒温恒湿柜里放了十年，你自己也可能想知道它被划伤那年到底放在哪儿。由于每次移动都带有日期，位置历史可以和借还、保养、品相变化一起，在@doc(copyHistory.readTimeline, "时间线")中对齐排列。

## 接下来可以做什么

- 在@doc(locations.setup)中构建一份值得信赖的存放位置地图。
- 副本离开你的手中是一次借出，而不是移动。参见@doc(loans.lendAndBorrow)。
- 在@doc(copyHistory.readTimeline)中查看移动记录在完整故事中的样子。
