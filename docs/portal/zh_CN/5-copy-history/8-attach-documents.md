# 为副本附加文档

收据、鉴定报告、证书、保单：一个记录详实的副本背后总跟着一堆单据。KolleK 把这些单据保存在副本本身上，或者更好的做法是，保存在它所属的那条具体记录上，这样收据就和它对应的交易放在一起，证书就和它对应的鉴定事件放在一起。

## 文档可以附加在哪里

一份文档可以附加到**副本本身**，也可以附加到它的任意一条历史记录上：

- **交易**（收据或发票）
- **估值**（书面鉴定报告）
- **保险记录**（保单条款）
- **借还**（签署的借还协议）
- **保养记录**（维修发票、前后对比照片）
- **来源事件**（证书、档案摘录）

请附加到最具体的位置。一张附加在对应购买交易上的收据不言自明；同一份文件如果松散地附加在副本上，日后的你就得去猜它是什么。只有那些不属于任何一条具体记录的单据，比如一组通用的品相照片，才附加到副本本身上。

## 文件还是链接

一份文档要么是一个**上传的文件**，要么是一个**外部链接**。

上传支持 PDF、常见图片格式（包括手机拍摄的 HEIC 照片）、Word 和 Excel 文件、CSV，以及纯文本，单个文件最大 12MB。存储的文件是私密的：只会提供给你账户中已登录的成员，永远不会通过公开地址访问。

外部链接指向存放在别处的内容，例如一个拍卖列表页面或制造商的档案条目。KolleK 只保存这个地址，不会保存页面本身的副本。

:::note
链接的页面可能会改变或消失，KolleK 只保存地址本身。对于任何你无法承受丢失的内容，请下载下来并作为文件上传，而不要只保存链接。
:::

## 附加一份文档

Priya 把她那块 1968 年欧米茄超霸表的拍卖收据附加到了对应的购买交易上。

::::steps
:::step title="打开副本的历史记录"
打开藏品，进入它的**历史**标签页，选择该副本，打开**文档**部分。

::screenshot{label="展开了文档部分的历史标签页"}
:::

:::step title="选择要附加到哪里"
添加一份文档，选择它归属的位置：副本本身，或它的某条记录。Priya 选择了那笔购买交易。
:::

:::step title="提供文件或链接"
上传**文件**，或者粘贴一个**外部链接**。
:::

:::step title="填写说明"
选择一个**文档类型**，填写**名称**（例如"拍卖收据，Fine Time Auctions"），并可以选填**描述**、**签发日期**和**参考编号**。保存。
:::
::::

## 文档类型

文档类型让一堆单据变得一目了然：**收据**、**发票**、**证书**、**鉴定报告**、**保险**、**照片**、**品相报告**、**修复报告**、**图录**、**往来信函**、**所有权记录**、**真伪记录**，以及**其他**。

:::note
文档不是藏品的照片。展示藏品外观的图片应该放在它的照片相册里（参见@doc(items.addPhotos)）。作为文档附加的照片是一种证据，例如一张和某次借还关联、带日期的品相照片。
:::
