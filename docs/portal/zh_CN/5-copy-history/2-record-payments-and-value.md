# 记录你的付款金额和它的价值

金钱和价值是收藏者最常问的两个问题，KolleK 特意把它们区分开来。**交易**记录的是实际发生的资金往来，**估值**记录的是某个副本在某个时间点值多少钱，不管有没有钱易手。本页将介绍如何记录这两者，并说明区分它们的规则。

如果你还没有读过@doc(copyHistory.concept, "副本历史详解")，请先阅读它。它介绍了这些记录是只增不改的历史，而不是可以随意覆盖的字段。

## 保持数据清晰的规则

购买价格是一笔交易，不是一次估值。

当 Priya 以 4200 元买下一块 1968 年的欧米茄超霸表时，这是一笔**购买**交易。它记录了她那天付了多少钱，而且这个记录永远不会改变。这块表*值多少钱*是另一个会随时间变化的问题，每一次的答案都是独立的一次估值。

KolleK 会根据这些记录自动推算出两个数字：

- 副本的**估计价值**取自最近一次估值的金额。没有任何估值记录的副本会显示为未估值，而不是价值为零。
- 副本的**购买价格**和**取得日期**取自最早的一笔取得性质的交易（购买、交换、收到赠予或继承）。

你永远不需要在副本上直接填写这些数字，只需要记录历史，当前的数字会自动根据历史算出。

## 记录一笔交易

交易涵盖围绕一个副本发生的任何金钱或所有权变动：买入、卖出、交换、支付费用，或运送到某地。

::::steps
:::step title="打开副本的历史记录"
打开藏品，进入它的**历史**标签页，选择你要处理的副本，然后打开**交易**部分。

::screenshot{label="展开了交易部分的历史标签页"}
:::

:::step title="添加一笔交易"
选择添加交易，并选取它的**类型**：购买、出售、交换、收到赠予、赠出、继承、退款、费用、税费、运费，或其他。
:::

:::step title="填写金额"
填写**金额**，并可以选填**税费**、**手续费**和**运费**，以便记录下真实的总成本，而不只是标价。
:::

:::step title="补充背景信息"
记录**交易对方**（你从谁那里买入或卖给了谁）、**日期**，以及**参考编号**，例如订单号或拍卖品编号。保存这笔交易。
:::
::::

Priya 记录了她购买超霸表的交易：类型为**购买**，金额 4200 元，拍卖行手续费 120 元，交易对方为 "Fine Time Auctions"，参考编号为拍品编号。这一条记录现在就回答了她付了多少钱、什么时候取得的，以及来自哪里。

:::note
最早的一笔取得性质的交易（购买、交换、收到赠予或继承）决定了副本的取得日期。没有这类交易的副本在统计数据中会算作日期未知，所以即使是很久以前购入的东西，也建议凭最佳猜测记录一个日期。
:::

## 记录一次估值

估值回答的是"这东西现在值多少钱，我有多大把握"。

::::steps
:::step title="打开估值部分"
在选中副本的同一个**历史**标签页中，打开**估值**部分。
:::

:::step title="添加一次估值"
选择一个**估值类型**：自行估算、专业鉴定、市场估价、保险价值、拍卖估价、自动估算，或其他。
:::

:::step title="填写价值和你的把握程度"
填写**金额**，选择**把握程度**（低、中、高，或未知），并记录**由谁估的值**。保存。

::screenshot{label="新建估值表单，包含类型、金额和把握程度"}
:::
::::

两年后，一位经销商告诉 Priya，这块超霸表现在能卖到大约 5500 元。她添加了一条新的估值：**市场估价**，5500 元，把握程度为**中**，估值人为该经销商。她原来的估值仍然保留在历史中，而副本的估计价值则更新为这个新数字。

:::note
重新估值总是写入一条新的估值记录，你永远不会去修改旧记录的数字，这样才能真实保留价值随时间变化的记录。这段历史正是统计页面中价值变化曲线的数据来源。
:::

## 这些数字会出现在哪里

你在这里记录的数字会反哺 KolleK 的其他部分：每个收藏上显示的总价值、@doc(insights.collectionStatistics, "收藏统计")里的价值变化和取得情况图表，以及按价值排序的热门藏品。详实的交易和估值记录，正是这些界面可信的基础。

## 接下来可以做什么

- 把相关单据和记录放在一起。参见@doc(copies.attachDocuments)，例如交易的收据或估值的鉴定报告。
- 想为这个价值投保？参见@doc(copies.insure)。
- 想建立完整的所有权故事？参见@doc(copies.traceProvenance)。
