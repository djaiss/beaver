# 三种角色的实际运作

本页是一份快速参考，说明每种角色实际能做什么。关于这个模型背后的理念，请先阅读@doc(accounts.usersAndRoles, "账户、用户与角色")。

简而言之：每个成员都可以查看，编辑者和所有者可以写入，只有所有者可以管理账户。

## 权限对照表

| 你想做的事 | 查看者 | 编辑者 | 所有者 |
| --- | --- | --- | --- |
| 浏览收藏、藏品和副本 | 是 | 是 | 是 |
| 查看统计数据、活动记录和仪表盘 | 是 | 是 | 是 |
| 创建和编辑收藏、藏品和副本 | 否 | 是 | 是 |
| 记录副本历史（交易、估值、借还等） | 否 | 是 | 是 |
| 上传照片和附加文档 | 否 | 是 | 是 |
| 设置收藏类型、自定义字段、标签和位置 | 否 | 是 | 是 |
| 使用回收站（恢复或永久删除） | 否 | 是 | 是 |
| 查看成员列表和待处理邀请 | 否 | 否 | 是 |
| 邀请成员、修改角色和移除成员 | 否 | 否 | 是 |
| 修改账户设置（名称、默认货币、入门清单） | 否 | 否 | 是 |
| 删除账户 | 否 | 否 | 是 |

## 如何理解这张表

**查看者是可靠的旁观者。** Leo 被邀请为查看者，可以浏览 Emma 的漫画，打开任意藏品，查看副本的历史记录，也可以欣赏统计数据。但他不能添加、修改或删除任何东西。查看者适合只想查阅信息的家庭成员，或者应该看到共享目录但不应拥有编辑权限的俱乐部成员。

**编辑者是实际动手的收藏者。** Sam 被邀请为编辑者，可以完成编目相关的一切工作：创建收藏、添加藏品和副本、记录历史、打标签、归档整理。编辑者适合正在积极维护目录的任何人。

**所有者同时也是管理员。** 除了编辑者能做的一切，所有者还负责管理成员和账户本身。每个账户至少需要一位所有者，KolleK 不允许最后一位所有者被降级或移除。

:::note
这些角色适用于整个账户，而不是单个收藏。收藏自身的可见性设置记录了它面向谁，一旦分享功能强制生效，可见性将决定谁能看到它，而角色则始终决定谁能修改它。参见@doc(sharing.overview, "可见性与分享")。
:::
