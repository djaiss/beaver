# 选择收藏的浏览方式

每个收藏都可以用三种方式浏览，KolleK 会记住你为每个收藏偏好的方式。本页介绍这几种布局、收藏页头部展示的信息，以及收藏内搜索的具体行为方式。

## 三种布局

打开一个收藏，用布局切换器在以下方式之间切换：

- **网格。** 以网格形式展示封面照片，最适合靠封面就能认出东西的视觉型收藏，这是默认布局。
- **列表。** 紧凑的行，一屏能看到更多藏品，适合快速浏览。
- **表格。** 每个细节一列，最适合需要在藏品之间比较具体信息的场景。

你的选择会**按用户、按收藏**分别记住。Emma 可以把她的漫画收藏看成网格，而同一个账户里的 Sam 却把同一个收藏看成表格，两人互不影响，也不会影响其他收藏的显示方式。

## 页头显示了什么

在收藏页面的顶部，你会看到它的**藏品数量**和**总价值**。总价值是收藏内每一件副本当前价值的总和，而每件副本的当前价值来自它最近一次的 @doc(copies.recordPaymentsAndValue, "估值")。从未估值过的副本不会计入其中，所以总价值偏低通常意味着缺少估值记录，而不是藏品不值钱。

## 在收藏内搜索

藏品列表上方的搜索框是一个快速过滤器，值得了解一下它到底做了什么、没做什么。

它会随着你输入，缩小**当前已加载在页面上的**藏品范围。一个收藏页面一次最多加载 1000 件藏品，所以对大多数收藏来说，这个过滤器实际上能覆盖全部内容。如果你的收藏超过这个数量，过滤器就只能看到已加载的那一部分。

:::note
这是一个当前视图内的过滤器，不是全局搜索。它只查看你当前打开的这个收藏，不会跨越你的整个账户。
:::

跨所有收藏的全局搜索正在规划中，目前还不可用，具体进展见 @doc(troubleshooting.featureStatus, "功能状态页面")。
