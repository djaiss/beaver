# 追踪你拥有的副本

一件藏品单独存在时只是一段描述。**副本**才是你实际拥有的某一个实物的记录，拥有它自己的品相、位置、状态和历史。本页介绍如何添加副本，以及副本上的每一个字段。

这种拆分背后的思路在 @doc(items.itemsVsCopies) 中有说明。添加副本需要**编辑者**或**所有者**角色。

## 添加一个副本

副本是直接在藏品表单上添加的，所以你可以在编目的同时记录它们。

::::steps
:::step title="打开藏品"
打开这件藏品并选择编辑它，然后添加一个**副本**。
:::

:::step title="记录它的实际状态"
从列表中选择**品相**，再选择存放它的**位置**。

::screenshot{label="副本行，品相和位置字段"}
:::

:::step title="设置状态和详情"
如果这是你实际拥有的东西，就把**状态**保留为“持有中”，或者选择其他状态。填写其余适用的字段，然后保存这件藏品。
:::
::::

同一件东西拥有两份？给同一件藏品添加第二个副本，而不是新建第二件藏品。每个副本都保留自己的品相、位置和历史。

## 副本字段详解

- **标识。** 序列号、评级编号，或任何能精确锁定这一个副本的标记。Priya 记录了她每只手表上刻的序列号。
- **@doc(conditions.overview, "品相")。** 这个副本的等级，从现成列表中选择（全新、几乎全新、已使用、有磨损、已损坏，以及你账户添加的其他品相）。
- **@doc(locations.overview, "位置")。** 副本目前存放的地方。之后通过移动来更改它会保留历史记录，见 @doc(copies.move, "移动一个副本")。
- **状态。** 这个副本目前处于生命周期的哪个阶段，见下方列表。
- **数量。** 用于完全相同、不需要逐个区分的副本，比如十包一模一样的未拆封补充包。如果每一件都需要单独记录，应该分别给它们各建一行。
- **处置日期。** 这个副本离开你手中的时间，用于“已售出”或“已处置”等状态。
- **备注。** 任何值得记住的、和这个副本本身相关的信息。
- **估计价值。** 一个快速估算这个副本值多少钱的数字。在后台，它会作为一条“自估”的 @doc(copies.recordPaymentsAndValue, "估值") 记录被保存，打开的是这个副本的价值历史，而不是直接写在副本本身上。对于你真正在意的副本，建议在那里添加带日期的正式估值。

## 状态的生命周期

- **持有中。** 在你手中，这是默认状态。
- **已订购。** 已购买但还未到货。
- **借出。** 在别人手里，但仍然属于你。变化的是保管权，而不是所有权，所以这个副本仍然计入你持有的数量。借出记录最好通过 @doc(loans.lendAndBorrow) 来完成，它会自动帮你设置这个状态。
- **已售出、已赠送。** 所有权转移给了别人。
- **已丢失、已被盗。** 在你不知情的情况下不见了。
- **已处置。** 被丢弃或回收了。
- **其他。** 列表未覆盖的任何情况。

“持有中”“已订购”和“借出”都算作“仍在你手中”，其余状态记录的则是已经离开你收藏、但你想保留其历史的副本。

## 钱去哪儿了

你可能注意到副本上没有“购买价格”这个字段，这是刻意的设计。你花了多少钱、什么时候买到的，来自它的**交易记录**；它随时间变化的价值，来自它的**估值记录**。这样保留的是完整的资金故事，而不是一个会被反复覆盖的单一数字。从 @doc(copies.recordPaymentsAndValue) 开始了解。
