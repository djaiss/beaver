# 动态与审计记录

KolleK 中的每一个操作都会被记录下来：谁做的、做了什么、什么时候做的。本页解释这些历史出现在哪两个地方，以及为什么你可以信任它。

在共享账户中，了解这一点尤其重要。当 Sam 修改了 Emma 账户中的一件藏品时，Emma 可以清楚地看到具体改了什么、是谁改的。

## 仪表盘上的动态

@doc(insights.dashboard, "仪表盘") 会展示整个账户中所有成员最近的操作，用带相对时间的简明句子回答“最近发生了什么”，是你外出一段时间后快速了解情况最快的方式。

## 藏品自身的动态记录

每件藏品都带有自己的一份记录，记录了它身上发生过的一切：创建、编辑、添加副本、上传照片，等等。当某次编辑改变了一个值时，记录会同时保存修改前和修改后的内容，所以你看到的不只是某个字段变了，还能看到它是从什么变成了什么。

打开一件藏品，找到它的动态记录，就能读到这个条目的完整故事。

## 为什么这份记录能够留存

审计记录会保存操作发生那一刻执行者的名字。如果某个成员之后离开了账户或删除了自己的用户，他们过去的操作依然会保留他们的名字。历史不会因为有人离开而变成空白。

:::note
审计记录会针对每一次改动自动写入，应用内没有任何方式可以编辑或删除它，这正是它值得信赖的原因。
:::

## 你自己的历史

每个用户在自己的个人资料中还拥有一份个人动态记录，只记录自己的操作。这个视图详见 @doc(activity.logAndSentEmails, "你的活动日志和已发送邮件")。
