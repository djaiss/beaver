# 自动删除不活跃用户

KolleK 提供了一项可自主开启的设置，会在你长期不活跃（大约六个月）后自动删除你的用户。这项功能是为那些不想让一个休眠的登录账户一直留在服务器上、而自己可能再也不会回来的人而准备的。

## 它是如何运作的

- 该设置**默认关闭，需要你自己开启**，并且只对你本人生效。没有人能替你开启它，你的选择也不会影响账户中的其他成员。
- 开启后，系统每天都会检查你最后一次活跃的时间。如果已经过去了大约六个月，你的用户就会被删除，效果与你@doc(users.deleteSelf, "自行删除")完全相同。
- 任何活动都会重置计时。只要登录一次就足够了。

## 开启或关闭

在个人资料的安全设置中找到该选项，选择是或否并保存。在删除真正触发之前，你随时可以改变主意。

:::warning
一旦触发不活跃删除，你的用户会被永久移除，登录信息将无法再使用，且不会再有额外的警告邮件。只有在你确实希望一个休眠账户自行消失时，才应该开启这项设置。
:::

## 这项设置适合你吗

如果你只是在试用 KolleK，或者对于一个被遗忘的登录账户感到介意，可以开启它。如果这份目录是一份长期档案，你可能一年都不会碰它，但仍希望它一直留着等你回来，就应该保持关闭。收藏是需要耐心的事，编目会话之间间隔八个月，在这个爱好里也是很正常的。

## 接下来去哪里

- 想主动、明确地删除用户？请参见 @doc(users.deleteSelf)。
- 查看你自己最近的活动，请参见@doc(activity.logAndSentEmails, "你的个人活动日志与已发送邮件")。
