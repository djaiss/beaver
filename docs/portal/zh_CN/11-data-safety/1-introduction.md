# 数据安全与日常维护

删除内容、恢复内容，以及分清什么是永久性的，这些事情永远不该靠猜测。本节会清楚说明哪些操作可以撤销、哪些不可以：回收站能接住什么、能保留多久，如何删除你自己的用户，删除账户究竟会摧毁什么，以及目前有哪些数据可迁移的方式。

请按需要阅读以下页面：

1. @doc(dataSafety.restoreFromTrash)。回收站能接住什么，能保留多久，以及如何恢复内容。
2. @doc(users.deleteSelf)。删除你自己，而不是整个账户。
3. @doc(users.inactiveDeletion)。可自主开启的自动清理设置。
4. @doc(accounts.delete)。产品中最具破坏性的操作。
5. @doc(dataSafety.backupCollectionData)。老实说明目前能导出哪些内容。

贯穿本节所有页面、值得牢记的一条原则是：如果某个操作即将变为永久性的，KolleK 会在你确认之前明确告知。如果拿不准，先去回收站看看，因为大多数日常删除操作都会先落到那里。
