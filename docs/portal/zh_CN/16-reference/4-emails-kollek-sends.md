# KolleK 发送的邮件

系统可能发送的每一种邮件，它们的触发条件，以及接收对象。你可以用本页来判断一封邮件是否合法，或者在你运行实例时用来核实邮件是否成功送达。

KolleK 会在你的@doc(activity.logAndSentEmails, "已发送邮件页面")中记录发给你的每一封邮件，包括送达和退信状态。尚未配置邮件服务的运营者应阅读 @doc(selfHosting.setupEmailDelivery)，因为一个全新的实例只会记录邮件，不会真正发送。

## 登录与保持登录

| 邮件 | 触发条件 | 接收对象 |
| --- | --- | --- |
| 账户邀请 | 所有者邀请某人加入账户。邀请链接七天后失效。 | 被邀请的邮箱地址 |
| 魔法链接 | 有人请求免密码登录链接。链接有效期为五分钟。 | 账户邮箱 |
| 邮箱验证 | 你注册账户，或修改了邮箱地址。 | 新邮箱地址 |
| 密码重置 | 你使用了忘记密码链接。重置链接有效期为 60 分钟。 | 账户邮箱 |

## 安全警报

当账户上发生重要情况时，这些邮件会主动发送给你。遇到意外邮件时该怎么处理，请参见 @doc(security.alertEmails)。

| 邮件 | 触发条件 | 接收对象 |
| --- | --- | --- |
| 登录失败提醒 | 已有账户的一次密码登录尝试失败。 | 账户邮箱 |
| 新登录提醒 | 一次成功登录发生，邮件中会说明所用设备。 | 账户邮箱 |
| IP 地址变更提醒 | 登录来源的 IP 地址与上次不同。 | 账户邮箱 |
| API 密钥创建提醒 | 你手动创建了一个 API 密钥。通过 API 登录方式创建的令牌不会触发此提醒。 | 账户邮箱 |
| API 密钥删除提醒 | 你删除了一个 API 密钥。 | 账户邮箱 |

## 发送给运营者的通知

这些邮件会发送到实例配置的运营者地址，而不是发送给收藏者。它们的作用是让运营服务器的人知道有人离开了。

| 邮件 | 触发条件 | 接收对象 |
| --- | --- | --- |
| 用户已删除 | 有人删除了自己的用户账户，邮件中包含其给出的原因。 | 运营者地址 |
| 用户被自动删除 | 系统删除了一个已开启不活跃自动删除、且已连续六个月不活跃的用户。 | 运营者地址 |

## 接下来去哪里

- 识别并应对安全警报：@doc(security.alertEmails)。
- 让你的实例真正能发送邮件：@doc(selfHosting.setupEmailDelivery)。
- 查看发给你的邮件记录：@doc(activity.logAndSentEmails, "你的个人活动日志与已发送邮件")。
