# 整理你的收藏目录

添加藏品只完成了一半的工作。本节介绍围绕藏品的组织结构：决定每件藏品要记录哪些信息的收藏类型和自定义字段，用来分组的分类、套装和系列，映射你实际存放位置的位置信息，横跨一切的标签，为你的副本评级的品相，以及让图片保持整洁的照片库。这些大多只需设置一次，之后随着收藏的增长再逐步完善即可。

按照与你当前要设置的内容相符的顺序阅读以下页面：

1. @doc(collectionTypes.setup)。定义每种藏品要记录哪些信息。
2. @doc(collectionTypes.importExport)。以 JSON 格式导入或导出一套类型定义。
3. @doc(categories.organizeItems)。在一个收藏内部建立嵌套的归档目录。
4. @doc(sets.trackCompletion)。了解你已拥有的和仍需收集的。
5. @doc(series.groupFranchise)。跨收藏关联同一系列作品的藏品。
6. @doc(locations.setup)。映射物品的实际存放位置。
7. @doc(tags.manageAccount)。维护共享的标签库。
8. @doc(conditions.manage)。维护副本可使用的品相等级列表。
9. @doc(photos.library)。在一处集中管理所有图片。

如果你还分不清分类、套装和系列之间的区别，建议先阅读概念页面 @doc(organizing.categoriesSetsAndSeries, "分类、套装和系列")，再看具体的操作指南。
