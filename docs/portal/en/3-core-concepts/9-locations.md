# Locations

A location answers the question every growing collection eventually asks: "where did I put it?" This page explains how KolleK models physical storage.

## What a location is

A location is a place where a @doc(items.itemsVsCopies, "copy") physically lives: a room, a shelf, a box, a safe. Each location can carry an emoji so it reads at a glance in lists.

Locations nest as deeply as you need, so they can mirror your real space. Noah models his: Living Room, then Shelf A inside it, then Crate 3 inside that. When he wonders where a record is, the answer is as precise as his map.

Locations are account wide. Define "Display Case" once and every collection can store copies there, which matches reality: one shelf can hold comics and coins side by side.

## Locations attach to copies, not items

An item is an idea, so it is the copy that sits somewhere. Emma's two copies of the same comic live in different places: one in Long box 1, one framed on the wall. Each copy points at its own current location.

A fresh account comes with a few starter locations (Living Room, Storage, Display Case, Garage, Office). Rename them, nest under them, or replace them with your own.

## Moves are remembered

When you move a copy, KolleK does not just overwrite the old location. It records the move, so the copy keeps a trail of everywhere it has been and when. The current location is simply the latest entry in that trail. This is part of @doc(copyHistory.concept, "a copy's history"), and the how to lives in @doc(copies.move).
