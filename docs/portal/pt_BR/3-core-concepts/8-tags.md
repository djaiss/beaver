# Tags

Tags são a forma mais leve de rotular itens. Sem hierarquia, sem estrutura, apenas palavras que você anexa e usa para filtrar depois. Esta página explica para que elas servem bem e em que se diferenciam das outras ferramentas de agrupamento.

## O que é uma tag

Uma tag é um rótulo curto e livre, como "Assinado", "Primeira edição" ou "Para segurar". Tags são compartilhadas por todas as coleções da conta: crie "Assinado" uma vez, e ela fica disponível tanto nos quadrinhos de Emma quanto nos vinis de Noah. Um item pode carregar quantas tags você quiser.

Tags se destacam para qualidades que atravessam todo o seu catálogo. "Assinado" se aplica a quadrinhos, livros e discos. Uma @doc(organizing.categoriesSetsAndSeries, "categoria") jamais conseguiria expressar isso, porque categorias vivem dentro de uma única coleção.

## Tags versus os outros agrupamentos

- **Categorias** arquivam um item em um único lugar dentro de uma coleção, e podem se aninhar. Tags não impõem hierarquia e alcançam tudo.
- **Sets** acompanham uma lista finita até a conclusão. Tags não contam nada.
- **Séries** reúnem uma franquia. Tags não carregam nenhum significado além da própria palavra.

Uma boa regra: se o rótulo responde "onde isso se arquiva", use uma categoria. Se ele marca uma qualidade ou um estado ("Assinado", "Duplicado", "À venda"), use uma tag.

## De onde vêm as tags

Qualquer pessoa com acesso de editor ou proprietário pode criar uma tag na hora, enquanto edita um item, ou escolher entre as já existentes. Proprietários e editores também podem organizar a lista compartilhada nas configurações da conta, abordado em @doc(tags.manageAccount).

## Excluindo uma tag

Excluir uma tag remove o rótulo de todo item que a carregava. Os itens em si permanecem intactos.

:::note
A exclusão de tag vale para a conta inteira. Remova "Assinado" e ela desaparece de todo item em todas as coleções de uma vez.
:::
