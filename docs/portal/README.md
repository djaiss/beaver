# KolleK documentation

Welcome. KolleK is a private, self hostable home for everything you collect: comics, vinyl, coins, watches, wine, books, cards, video games, or anything else you can own. You create collections, describe them with your own custom fields, add items with photos and tags, and track every physical copy you hold, including its condition, storage location, value, and full history.

This portal is organized so a new reader can start at the top and never meet an advanced idea before they need it.

## Where should you start

**If you collect things.** Read [What is KolleK](getting-started/what-is-kollek.md), [create your account](getting-started/create-your-account.md), follow [the getting started checklist](getting-started/getting-started-checklist.md), then learn [how KolleK is organized](core-concepts/how-kollek-is-organized.md) and the key idea of [items versus copies](core-concepts/items-and-copies.md). From there, the [Core Features](core-features/README.md) guides carry you through daily cataloguing, and [Tracking a Copy's History](copy-history/README.md) is waiting when you want to record value, loans, and provenance.

**If you own an account others use.** Everything a collector reads, plus [accounts, users, and roles](core-concepts/accounts-users-and-roles.md), the [Collaboration](collaboration/README.md) section, [account settings](account-and-profile/account-settings.md), and [Data Safety](data-safety/README.md).

**If you run an instance.** Start with [Cloud version versus self hosting](getting-started/cloud-vs-self-hosting.md), then work through [Self Hosting and Administration](self-hosting/README.md) from the [overview](self-hosting/README.md) onward, paying particular attention to [the application key](self-hosting/application-key-and-encryption.md).

**If you are building against the API.** Read the [API overview](developers/api-overview.md) and [authenticate with the API](developers/authenticate-with-the-api.md), then use the generated endpoint reference at `/docs/api`. Read [webhooks](developers/webhooks.md) knowing that no event fires them yet.
