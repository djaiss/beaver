# カテゴリ、セット、シリーズ

KolleKには、アイテムをグループ化する3つの方法があり、それぞれが答える問いを知るまでは似たように見えます。このページではそれらを整理し、常に正しいものを選べるようにします。

## 3つの問い

- **このアイテムはどこにファイリングされるか。** それは**カテゴリ**です。
- **完成させようとしているのはどの有限のリストか。** それは**セット**です。
- **収集しているすべてのものの中で、このアイテムはどのフランチャイズに属するか。** それは**シリーズ**です。

## カテゴリ: 1つのコレクション内でのファイリング

カテゴリは1つの@doc(collections.overview, "コレクション")内でアイテムをファイリングするもので、入れ子にできます。Emmaの「My Comics」コレクションにはMarvelカテゴリがあり、その中にSpider-ManとX-Menが入れ子になっています。カテゴリを閲覧すると、そのカテゴリのアイテムだけが、独自の件数と統計とともに表示されます。

カテゴリは自分のコレクションに属し、その外に出ることはできません。カテゴリを削除しても、その中のアイテムが削除されることはなく、単にカテゴリなしの状態になります。

## セット: 完成を目指す有限のリスト

セットは、終わりのあるチェックリストです。Emmaがあるシリーズの最初の10号を集めたい場合、**目標数**を10に設定してセットを作成します。セットにアイテムを追加していくと、KolleKは所有数と目標数を表示し、その完成度は彼女の@doc(insights.collectionStatistics, "コレクション統計")に反映されます。

カテゴリと同様、セットも1つのコレクションに属します。目標数が0より大きいセットだけが完成度の統計に反映され、目標数のないセットは単なる名前付きのグループです。

## シリーズ: コレクションをまたぐフランチャイズ

シリーズは、どこにあっても、1つのフランチャイズに属するすべてを結びつけます。ハリー・ポッターは、Emmaの本、映画、LEGOという3つの異なるコレクションにまたがっています。シリーズはそのすべてをまたいで存在し、そのページでは紐づいたアイテムが、それぞれが属するコレクションごとにグループ化され、いくつのコレクションにまたがっているかも表示されます。

シリーズは完成度を追跡しません。「ハリー・ポッターに関するものをすべて見せて」という問いに答えるものであり、「まだ完成していないか」という問いには答えません。

## 比較表

| | カテゴリ | セット | シリーズ |
|---|---|---|---|
| 範囲 | 1つのコレクション | 1つのコレクション | アカウント全体 |
| 入れ子 | できる | できない | できない |
| 完成度の追跡 | しない | する(目標数付き) | しない |
| 答える問い | どこにファイリングされるか | どのリストの一部か | どのフランチャイズか |

1つのアイテムがこの3つすべてを同時に使うこともできます。Marvelにファイリングされ、「First ten issues」セットの一部であり、Spider-Manシリーズにリンクされている、というように。

これらよりもさらに軽量な、構造をまったく持たない横断的なラベルが欲しい場合は、それが@doc(tags.overview, "タグ")の役割です。
