# 基本コンセプト

KolleKには独自のデータモデルがあり、それを理解するために20分ほど時間をかける価値があります。このセクションにあるいくつかの考え方さえ理解すれば、以降のすべてのタスクページが自然に理解できるようになります。まずは早い段階で一度これらのページを読み、用語があいまいになったときはまた戻ってきてください。

ページは次の順番で読んでください。

1. @doc(kollek.howOrganized)。全体の考え方を1ページにまとめたものです。
2. @doc(accounts.usersAndRoles)。ワークスペースと、そこに参加する人、それぞれができることです。
3. @doc(collections.overview)。カタログ化するすべてを保持する、名前付きのグループです。
4. @doc(items.itemsVsCopies)。KolleKの中で最も重要な考え方です。
5. @doc(collectionTypes.overview)。アイテムの種類ごとに何を記録するかを決める方法です。
6. @doc(organizing.categoriesSetsAndSeries)。それぞれ異なる問いに答える、3つのグループ分けツールです。
7. @doc(tags.overview)。アイテムをまたいでラベル付けし、絞り込むための最も軽量な方法です。
8. @doc(locations.overview)。物理的な保管場所のモデル化の方法です。
9. @doc(conditions.overview)。コピーの状態を評価する方法です。
10. @doc(copyHistory.concept)。価値、移動、来歴がどこに記録されるかです。
11. @doc(sharing.overview)。コレクションを誰が、いつ見られるかです。
12. @doc(dataSafety.howProtected)。暗号化、監査ログ、そしてその限界です。

時間が5分しかない場合は、@doc(items.itemsVsCopies)を読んでください。以降のドキュメント全体が常に前提とする、唯一のページです。
