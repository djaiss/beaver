# コピーの履歴を記録する

所有している@doc(items.itemsVsCopies, "コピー")には、それぞれの物語をすべて記録できます。いくら支払ったか、いくらの価値があるか、誰が保険をかけたか、誰が借りているか、どんな手入れを受けてきたか、どこから来て、どこに保管されてきたか。その物語の各部分は日付付きの独立した記録であり、このセクションではそれぞれの作成方法を説明します。

これらのページはすべて同じ出発点を共有しています。アイテムを開き、**履歴**タブに移動して、作業したいコピーを選びます。前提条件も共通です。まだ@doc(copyHistory.concept, "コピーの履歴の仕組み")を読んでいない場合は、まずそちらを読んでください。記録がなぜこのような仕組みになっているかが説明されています。

記録の追加や変更には編集者またはオーナーの権限が必要です。閲覧者はすべてを読めますが、何も変更できません。

ページはどの順番で読んでも構いませんし、購入時からのコピーの物語をたどることもできます。

1. @doc(copies.recordPaymentsAndValue)。金額の取引と、価値の評価。
2. @doc(copies.insure)。対象物のそばに保険の詳細を残します。
3. @doc(loans.lendAndBorrow)。所有権を見失わずに、預けている相手を追跡します。
4. @doc(copies.recordMaintenance)。清掃、メンテナンス、修復を記録します。
5. @doc(copies.traceProvenance)。所有履歴と真贋の裏付けを積み上げます。
6. @doc(copies.move)。以前の保管場所を残したまま、現在の保管場所を変更します。
7. @doc(copies.attachDocuments)。領収書、証明書、鑑定書をコピーに紐づけて保管します。
8. @doc(copyHistory.readTimeline)。すべての記録が一つの時系列にまとまった様子を確認します。

一つだけ記録するなら、取引と評価を記録してください。これらは、@doc(insights.collectionStatistics, "コレクション統計")を含む、KolleK全体の価値の数値を支えています。
