# KolleKドキュメント

ようこそ。KolleKは、コミック、レコード、コイン、腕時計、ワイン、本、カード、ビデオゲーム、その他何でも、あなたが収集するあらゆるもののための、プライベートでセルフホスト可能な場所です。コレクションを作成し、独自のカスタムフィールドで説明を加え、写真とタグを付けてアイテムを追加し、コンディションや保管場所、価値、履歴も含めて、所有する一つひとつのコピーを管理できます。

このポータルは、はじめて読む方が最初から順番に目を通せば、必要になる前に高度な内容に出会わないように構成されています。

## 各セクション

1. @doc(gettingStarted.index)。KolleKとは何か、アカウントの作成方法、最初のアイテムの登録方法について。
2. @doc(coreConcepts.index)。基本的な考え方: アカウント、コレクション、アイテム、コピー、そしてそれらを取り巻く概念について。
3. @doc(coreFeatures.index)。日々のカタログ作成: コレクション、アイテム、コピー、写真、タグについて。
4. @doc(copyHistory.index)。金額、価値、保険、貸出、手入れ、来歴、移動、書類について。
5. @doc(organizing.index)。コレクションタイプとカスタムフィールド、カテゴリ、セット、シリーズ、保管場所、タグ、写真ライブラリについて。
6. @doc(insights.index)。統計画面とダッシュボードについて。
7. @doc(collaboration.index)。招待、権限、コレクションの共有について。
8. @doc(accountAndProfile.index)。個人情報、言語、アバター、アカウント設定について。
9. @doc(security.index, "セキュリティ")。2段階認証、リカバリーコード、マジックリンク、パスワード、APIキーについて。
10. @doc(dataSafety.index)。ゴミ箱、削除、データのエクスポートについて。
11. @doc(tutorials.index)。各機能を組み合わせた、最初から最後までの完全な流れについて。
12. @doc(developers.index)。KolleKと連携する開発者向けの、認証、規約、Webhookについて。
13. @doc(selfHosting.index, "セルフホスティングと管理")。インスタンスのインストール、設定、アップグレード、バックアップ、管理について。
14. @doc(troubleshooting.index)。想定通りに動作しないときの、すぐに使える回答集。
15. @doc(reference.index)。用語集、各フィールドとステータスの選択肢、KolleKが送信するすべてのメールについて。

## どこから始めればよいか

**物を収集している方は。** @doc(kollek.whatIs)、@doc(accounts.create, "アカウントを作成")を読み、@doc(gettingStarted.checklist, "はじめにチェックリスト")に沿って進めた後、@doc(kollek.howOrganized, "KolleKの構成")と@doc(items.itemsVsCopies, "アイテムとコピーの違い")という重要な考え方を学んでください。そこから先は、@doc(coreFeatures.index)のガイドが日々のカタログ作成を案内してくれます。価値や貸出、来歴を記録したくなったときは、@doc(copyHistory.index)が役立ちます。

**他の人も使うアカウントを所有している方は。** コレクターが読む内容すべてに加えて、@doc(accounts.usersAndRoles, "アカウント、ユーザー、権限")、@doc(collaboration.index)のセクション、@doc(accounts.settings, "アカウント設定")、@doc(dataSafety.index, "データの安全性")を読んでください。

**インスタンスを運用している方は。** まず@doc(kollek.hostingOptions)から始め、@doc(selfHosting.index, "概要")から@doc(selfHosting.index, "セルフホスティングと管理")を順に読み進めてください。特に@doc(selfHosting.applicationKeyAndEncryption, "アプリケーションキー")には注意してください。

**APIを使って開発している方は。** @doc(api.overview)と@doc(api.authenticate, "APIで認証する")を読んでから、`/docs/api`に生成されたエンドポイントリファレンスを参照してください。@doc(webhooks.overview, "Webhook")も読んでおいてください。ただし、現時点ではそれらを発火させるイベントはまだありません。
