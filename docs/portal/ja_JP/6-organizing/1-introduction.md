# カタログの整理

アイテムを追加するだけでは半分の作業に過ぎません。このセクションでは、アイテムを取り巻く構造について説明します。各アイテムが何を記録するかを決めるコレクションタイプとカスタムフィールド、アイテムをまとめるカテゴリ、セット、シリーズ、物理的な保管場所を表す保管場所、すべてを横断するタグ、コピーを格付けするコンディション、そして画像を整理する写真ライブラリです。これらのほとんどは一度設定すれば、あとはコレクションの成長に合わせて調整していくだけです。

セットアップしたい内容に合わせて、好きな順番でページを読み進めてください。

1. @doc(collectionTypes.setup)。各アイテムの種類ごとに何を記録するかを定義します。
2. @doc(collectionTypes.importExport)。タイプ定義をJSONとして書き出したり読み込んだりします。
3. @doc(categories.organizeItems)。コレクション内に入れ子構造の整理ツリーを作成します。
4. @doc(sets.trackCompletion)。何を持っていて、何がまだ必要かを把握します。
5. @doc(series.groupFranchise)。複数のコレクションをまたいでアイテムを結び付けます。
6. @doc(locations.setup)。物のある場所を地図のように整理します。
7. @doc(tags.manageAccount)。アカウント共有のタグ一覧を整えます。
8. @doc(conditions.manage)。コピーに付けられるコンディションの一覧を整えます。
9. @doc(photos.library)。すべての画像を一箇所で扱います。

カテゴリ、セット、シリーズの違いがはっきりしない場合は、操作方法のガイドに進む前に @doc(organizing.categoriesSetsAndSeries, "カテゴリ、セット、シリーズ") の概念ページから読むとよいでしょう。
