# セットのコンプリート状況を管理する

@doc(organizing.categoriesSetsAndSeries, "セット") は、1つのコレクションの中でコンプリートを目指す有限のリストです。全12号の限定シリーズ、ルーキーカードのフルチーム、ディスコグラフィーなどがこれにあたります。セットに目標数を設定すれば、KolleKが所有数と目標数を対比して表示してくれるので、「あとどれくらい足りないか」に常に答えられます。

セットを作成・変更するには編集者またはオーナーの権限が必要です。

## セットを作成する

::::steps
:::step title="コレクションのセットを開く"
コレクションを開き、**セット**に移動します。
:::

:::step title="セットを作成する"
**新しいセット**を選び、名前を付け、**目標数**(完全なセットが持つアイテムの数)を入力します。Noahは目標数100で「Blue Note 1500シリーズ」を作成します。

::screenshot{label="目標数フィールドがある新規セットフォーム"}
:::

:::step title="アイテムを追加する"
アイテムを追加・編集する際に、アイテムのフォームでセットを選びます。セットに分類したアイテムごとに、所有数が増えていきます。
:::
::::

## コンプリート状況の数え方

セットのページでは、所有数を目標数に対して集計します。100枚のシリーズのうち37枚を所有していれば、「37/100」と表示されます。コンプリート状況は @doc(insights.collectionStatistics, "コレクション統計") にも反映され、すべてのセットが合算されて全体のコンプリート率になります。

知っておくべき点が2つあります。

- **コンプリート統計に反映されるのは、目標数が0より大きいセットだけです。** 目標数のないセットもアイテムをグループ化しますが、コンプリート状況としては扱われません。
- **目標数を超えてアイテムを持っていても、コンプリート(100%)として扱われ、それ以上にはなりません。** ダブりやバリアントによって100%を超えることはありません。

## セットとシリーズの違い

セットは1つのコレクションの中に存在し、コンプリート状況を追跡します。@doc(series.groupFranchise, "シリーズ") は複数のコレクションをまたぎ、コンプリート状況は追跡しません。フランチャイズをまとめるだけの存在です。「あとどれが足りないか」を知りたいなら、セットを使いましょう。

## セットを削除する

セットを削除すると @doc(dataSafety.restoreFromTrash, "ゴミ箱") に移動し、そこから復元できます。

:::warning
セットを削除すると、グループ化とそのコンプリート状況の追跡も失われます。セット内のアイテム自体は削除されず、すべてのデータを保持したままです。
:::

## 次に読むページ

- 複数のコレクションにまたがるフランチャイズをまとめるには @doc(series.groupFranchise, "シリーズ") へ。
- コンプリート状況の集計を見るには @doc(insights.collectionStatistics) へ。
