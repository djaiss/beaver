# コレクションの作成と管理

@doc(collections.overview, "コレクション") は、他のあらゆるものが収まる器なので、通常はまずこれを作成するところから始まります。このページでは、コレクションの作成方法、フォーム上のすべての項目、あとからの編集方法、そして削除すると実際に何が起こるかを説明します。

## 誰が操作できるか

コレクションの作成、編集、削除には **編集者** または **オーナー** の @doc(accounts.usersAndRoles, "権限") が必要です。閲覧者はコレクションを閲覧できますが、作成や変更はできません。

## コレクションを作成する

Noahはレコードのカタログを作り始めようとしています。手順は次のとおりです。

::::steps
:::step title="新しいコレクションを開始する"
コレクション画面から**新しいコレクション**を選びます。

::screenshot{label="コレクション画面の新しいコレクションボタン"}
:::

:::step title="名前と説明を入力する"
「レコード」のような**名前**を付け、任意で短い**説明**と、一覧で目立たせるための**絵文字**を設定します。
:::

:::step title="コレクションタイプを選ぶ"
どの @doc(collectionTypes.overview, "コレクションタイプ") を適用するかを選びます。Noahは用意されている**レコード**タイプを選び、アイテムにアーティスト、アルバム、プレスといったフィールドを持たせます。複数のタイプを有効化することも、まったく有効化しないこともでき、あとから変更もできます。
:::

:::step title="通貨と公開範囲を設定する"
このコレクションで使う金額の**通貨**と、その**公開範囲**を選びます。迷った場合はデフォルトのままにしておきましょう。プライベートが安全な出発点です。

::screenshot{label="コレクションフォームの通貨と公開範囲フィールド"}
:::

:::step title="保存する"
コレクションを保存します。一覧に表示され、最初のアイテムを迎える準備が整います。
:::
::::

## 各項目の説明

- **名前。** あらゆる場所でコレクションを表す名前です。必須です。
- **説明。** 何が収められているかを一文で説明します。任意ですが、コレクションが増えてくると役立ちます。
- **絵文字。** 12種類の固定パレット(📦 📚 💿 🃏 🍷 🎮 🧸 🪙 🖼️ ⌚ 👟 📷)から選べる、見た目の目印です。任意です。
- **コレクションタイプ。** 有効化したタイプによって、このコレクション内のアイテムがどのカスタムフィールドを記録できるかが決まります。複数有効化することもでき、たとえば「読書」という1つのコレクションにコミックと本の両方を含めることもできます。
- **通貨。** このコレクション内のすべての金額(値、統計)がこの通貨で扱われます。18種類の通貨から選べます。アカウントのデフォルト通貨と異なる通貨を設定することもでき、たとえばワインはユーロで、それ以外はドルで購入している場合などに便利です。
- **公開範囲。** そのコレクションを誰向けに公開するかです。**プライベート**(自分のみ)、**共有**(アカウント内の全員)、**公開**(リンクを知っている誰でも閲覧可能、読み取り専用)から選べます。この設定は現時点では記録されるのみで、共有機能が実装され次第、実際に適用されるようになります。@doc(sharing.overview, "公開範囲と共有") の概念ページでこのモデルと現在の状況を説明しており、@doc(collections.share) で変更方法を解説しています。

## コレクションを編集する

コレクションを開き、編集を選びます。同じフォームが同じ項目とともに表示され、いつでもどの項目も変更できます。コレクションの名前を変更したり絵文字を切り替えたりしても、中身には何も影響しません。

## コレクションを削除する

コレクションを開き、削除を選んで確定します。

:::warning
コレクションを削除すると、その中のすべてのアイテムもゴミ箱に送られます。コレクションとそのアイテムは、一定期間(デフォルトで30日間)ゴミ箱に残り、その後完全に削除されます。
:::

ゴミ箱にある間は、まだ考え直すことができます。@doc(dataSafety.restoreFromTrash) を参照してください。
