# アカウントを削除する

アカウントの削除は、KolleKの中で最も破壊的な操作です。ワークスペース全体、つまりすべてのコレクション、すべてのアイテム、履歴を含むすべてのコピー、すべての写真と書類、そしてすべてのメンバーのアクセス権が失われます。実行できるのは@doc(accounts.usersAndRoles, "オーナー")だけです。

:::warning
アカウントの削除は取り消せません。何もゴミ箱には入らず、何も復元できず、インスタンスの運用者を含め誰も元に戻すことはできません。すべてのメンバーが一瞬ですべてを失います。
:::

## 削除する前に

落ち着いて、次の3点を確認してください。

- **本当に望んでいるのはこれですか。それとも@doc(users.deleteSelf, "自分のユーザーを削除する")ことですか。** 共有アカウントから抜けるだけなら、自分を削除するだけで十分です。アカウントとカタログは、あなたがいなくても残ります。
- **他の誰かがこのアカウントに依存していませんか。** 確定した瞬間、アカウントのすべてのメンバーがアクセス権とデータを失います。先に伝えておいてください。
- **必要なものはすでに取り出しましたか。** 残しておきたい@doc(collectionTypes.importExport, "コレクションタイプの定義")はエクスポートしておいてください。セルフホストのインスタンスであれば、@doc(selfHosting.backupAndRestore)で説明されている手順で、先に完全なバックアップを取ってください。削除後には、バックアップを取る対象すら残っていません。

## アカウントを削除する

**アカウント設定**から、危険な操作の欄にある削除オプションを見つけ、確定します。アカウントとその中身はすべて削除され、すべてのメンバーは完全にサインアウトされます。

## 削除後に残らないもの

すべてです。コレクション、アイテム、コピー、カテゴリ、セット、シリーズ、タグ、保管場所、カスタムタイプとフィールド、写真、書類、コピーの履歴のすべて、アクティビティの記録、すべてのメンバー、保留中のあらゆる招待。関係していたメールアドレスは新しいアカウントを登録できる状態に戻りますが、そのアカウントは空の状態から始まります。
