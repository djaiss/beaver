# データの安全性とメンテナンス

何かを削除すること、それを復元すること、そして何が取り返しのつかない操作なのかを知ることは、決して当てずっぽうであってはなりません。このセクションでは、元に戻せることと戻せないことをはっきりさせます。ゴミ箱が何をどれくらいの期間保持するか、自分のユーザーを削除する方法、アカウントを削除すると実際に何が失われるのか、そして現時点でどのようなデータの持ち出しができるのかです。

必要になったときに、それぞれのページを読んでください。

1. @doc(dataSafety.restoreFromTrash)。ゴミ箱が何を受け取り、どれくらいの期間保持し、どうやって元に戻すか。
2. @doc(users.deleteSelf)。アカウント全体ではなく、自分自身を削除する方法。
3. @doc(users.inactiveDeletion)。任意で有効にできる、自動的なクリーンアップ設定。
4. @doc(accounts.delete)。この製品の中で最も破壊的な操作。
5. @doc(dataSafety.backupCollectionData)。現時点で何がエクスポートできるのか、正直なところ。

このセクション全体を通して持ち帰ってほしいルールは1つです。何かが取り返しのつかない操作になろうとしているときは、確定させる前にKolleKが必ずそう伝えます。迷ったら、まずゴミ箱を確認してください。日常的な削除の多くはそこに残っています。
