# セキュリティ概要

KolleKには、あなたにとって大切な記録が保存されています。何を所有しているか、それがいくらの価値があるか、どこに保管しているかです。このページでは、あなたのユーザーとデータを守るための各種機能をまとめています。どの機能を有効にするかはあなた次第です。すべて任意ですが、多くはわずか5分で設定できる価値があります。

## パスワード

すべてのアカウントはパスワードから始まります。KolleKはパスワードを設定する際に2つのルールを適用します。8文字以上であること、そして過去の情報漏洩で流出したことが知られているパスワードのリストと照合されることです。設定しようとしたパスワードが拒否された場合は、それがそうしたリストに含まれていたためなので、他では使ったことのないものを選んでください。

パスワードはいつでも変更でき、忘れた場合もアクセスを取り戻せます。@doc(auth.resetPassword) を参照してください。

## 2段階認証

もっとも効果の大きい強化策です。2段階認証を有効にすると、パスワードでのサインインに加えて、スマートフォンの認証アプリが生成する6桁のコードが求められるようになります。パスワードが盗まれただけでは、もうログインできません。

@doc(security.twoFactorAuth) で設定方法を確認し、利用する前に @doc(security.recoveryCodes, "リカバリーコード") について理解しておいてください。

## リカバリーコード

2段階認証を有効にすると、KolleKは8個のリカバリーコードを発行します。それぞれ一度だけ使用でき、スマートフォンを紛失した際に認証アプリのコードの代わりとして使ってログインできます。安全な場所に保管してください。詳しくは @doc(security.recoveryCodes) を参照してください。

## マジックリンク

パスワードを使わずにサインインする方法です。KolleKから届くリンクをクリックするだけで直接サインインでき、有効期限は5分間です。便利な反面、理解しておくべき点が一つあります。マジックリンクは2段階認証のコードを求めません。受信箱にアクセスできること自体が、すでに第二の要素として機能しているためです。使いどころについては @doc(auth.magicLinks) を参照してください。

## APIキー

KolleKのAPIを利用する場合は、個人用のAPIキーで認証します。APIキーはプロフィールから作成・失効でき、作成または削除されるたびにKolleKからメールが届くため、自分が作成していないキーを見逃すことはありません。詳しくは @doc(apiKeys.manage) を参照してください。

## アラートメール

KolleKは、お知らせすべきイベントを監視しています。サインイン失敗、新しいデバイスからのサインイン、IPアドレスの変化、APIキーの作成や削除などです。これらが発生すると、メールが届きます。各アラートの意味と対応方法については @doc(security.alertEmails) を参照してください。

## おすすめの設定

2つだけ設定するなら、次を行ってください。

1. @doc(security.twoFactorAuth, "2段階認証") を有効にする。
2. @doc(security.recoveryCodes, "リカバリーコード") をスマートフォン以外の場所に保存する。

このセクションのその他の内容は、必要になったときで構いません。
