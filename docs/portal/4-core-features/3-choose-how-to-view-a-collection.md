# Choose how to view a collection

Every collection can be browsed three ways, and KolleK remembers which way you prefer for each one. This page covers the layouts, what the collection header tells you, and exactly how searching inside a collection behaves.

## The three layouts

Open a collection and use the layout switcher to move between:

- **Grid.** Cover photos in a grid. Best for visual collections, where you recognize things by their cover. This is the default.
- **List.** Compact rows, more items on screen. Good for quick scanning.
- **Table.** A column per detail. Best when you care about comparing details across items.

Your choice is remembered **per user, per collection**. Emma can browse her comics as a grid while Sam, in the same account, sees the same collection as a table. Neither affects the other, and neither affects other collections.

## What the header shows

At the top of the collection you will see its **item count** and its **total value**. The total value is the sum of the current value of every copy inside, where each copy's current value comes from its most recent [valuation](../5-copy-history/2-record-payments-and-value.md). Copies that have never been valued contribute nothing, so a low total usually means missing valuations, not worthless items.

## Searching inside a collection

The search box above the items is a fast filter, and it is worth understanding what it does and does not do.

It narrows the items **currently loaded on the page** as you type. A collection page loads up to 1000 items at once, so for most collections the filter effectively covers everything. If your collection is larger than that, the filter only sees the loaded page.

:::note
This is an in view filter, not a global search. It looks at the collection you have open, not across your whole account.
:::

A global search across every collection is planned but not available yet. See the [feature status page](../15-troubleshooting/5-feature-status.md) for where that stands.
