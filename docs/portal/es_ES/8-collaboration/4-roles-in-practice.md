# Los tres roles en la práctica

Esta página es una referencia rápida de lo que cada rol puede hacer en realidad. Para las ideas detrás del modelo, lee primero @doc(accounts.usersAndRoles, "cuentas, usuarios y roles").

La versión corta: todo miembro puede leer, los editores y propietarios pueden escribir, y solo los propietarios pueden administrar la cuenta.

## Tabla de capacidades

| Lo que quieres hacer | Lector | Editor | Propietario |
| --- | --- | --- | --- |
| Explorar colecciones, elementos y ejemplares | Sí | Sí | Sí |
| Ver estadísticas, actividad y el panel | Sí | Sí | Sí |
| Crear y editar colecciones, elementos y ejemplares | No | Sí | Sí |
| Registrar el historial de un ejemplar (transacciones, valoraciones, préstamos y el resto) | No | Sí | Sí |
| Subir fotos y adjuntar documentos | No | Sí | Sí |
| Configurar tipos de colección, campos personalizados, etiquetas y ubicaciones | No | Sí | Sí |
| Usar la papelera (restaurar o eliminar permanentemente) | No | Sí | Sí |
| Ver la lista de miembros y las invitaciones pendientes | No | No | Sí |
| Invitar, cambiar roles y eliminar miembros | No | No | Sí |
| Cambiar la configuración de la cuenta (nombre, moneda predeterminada, lista de comprobación) | No | No | Sí |
| Eliminar la cuenta | No | No | Sí |

## Cómo interpretar esto

**Los lectores son ojos de confianza.** Leo, invitado como lector, puede explorar los cómics de Emma, abrir cualquier elemento, leer el historial de un ejemplar y admirar las estadísticas. No puede añadir, cambiar ni eliminar nada. Los lectores encajan bien con familiares que quieren consultar cosas, o miembros de un club que deberían ver el catálogo compartido sin permisos de edición.

**Los editores son coleccionistas en acción.** Sam, invitado como editor, puede hacer todo lo relacionado con catalogar: crear colecciones, añadir elementos y ejemplares, registrar historial, etiquetar, clasificar y organizar. Los editores encajan bien con cualquiera que mantenga el catálogo activamente.

**Los propietarios también son administradores.** Además de todo lo que puede hacer un editor, los propietarios gestionan a las personas y la cuenta misma. Toda cuenta necesita al menos uno, y KolleK no permitirá que se degrade o elimine al último propietario.

:::note
Estos roles se aplican a toda tu cuenta, no a colecciones individuales. El ajuste de visibilidad propio de una colección registra a quién está destinada, y en cuanto se aplique la función de compartir, decidirá quién puede verla, mientras que los roles siguen decidiendo quién puede cambiarla. Ver @doc(sharing.overview, "visibilidad y compartir").
:::
