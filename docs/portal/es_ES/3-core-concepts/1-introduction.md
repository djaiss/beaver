# Conceptos básicos

KolleK tiene un modelo de datos concreto, y merece la pena dedicarle veinte minutos. Una vez que entiendas el puñado de ideas de esta sección, todas las páginas de tareas del resto de la documentación resultarán evidentes. Lee estas páginas una vez, pronto, y vuelve a ellas cada vez que un término te resulte confuso.

Lee las páginas en este orden:

1. @doc(kollek.howOrganized). Todo el modelo mental en una sola página.
2. @doc(accounts.usersAndRoles). El espacio de trabajo, las personas que hay en él y qué puede hacer cada una.
3. @doc(collections.overview). Los grupos con nombre que contienen todo lo que catalogas.
4. @doc(items.itemsVsCopies). La idea más importante de KolleK.
5. @doc(collectionTypes.overview). Cómo decides qué registra cada tipo de elemento.
6. @doc(organizing.categoriesSetsAndSeries). Tres herramientas de agrupación, cada una responde a una pregunta distinta.
7. @doc(tags.overview). La forma más ligera de etiquetar y organizar elementos.
8. @doc(locations.overview). Cómo se modela el almacenamiento físico.
9. @doc(conditions.overview). Cómo se califica el estado de un ejemplar.
10. @doc(copyHistory.concept). Dónde viven el valor, el movimiento y la procedencia.
11. @doc(sharing.overview). Quién puede ver una colección, y cuándo.
12. @doc(dataSafety.howProtected). Cifrado, el registro de auditoría y sus límites.

Si solo tienes cinco minutos, lee @doc(items.itemsVsCopies). Es la página en la que se apoya constantemente el resto de la documentación.
