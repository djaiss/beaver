# Reference

The pages in this section are lookup material. They are meant to be scanned, not read top to bottom. When another page mentions a term, an option, or an email you want the full picture of, the answer is here.

1. @doc(reference.glossary). Every product term, defined in a line or two, with a link to the page that explains it fully.
2. @doc(reference.fieldAndStatus). Every option set you meet in a form, from copy statuses to provenance event types, with a one line meaning for each.
3. @doc(reference.emailsSent). Every email the system can send, what triggers it, and who receives it.

If you are new to KolleK, do not start here. Start with @doc(gettingStarted.index), and come back when you need to look something up.
