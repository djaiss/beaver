# Kategorien, Sets und Reihen

KolleK gibt dir drei Wege, Objekte zu gruppieren, und sie sehen sich ähnlich, bis du die Frage kennst, die jeder beantwortet. Diese Seite entwirrt sie, damit du jedes Mal zum richtigen Werkzeug greifst.

## Die drei Fragen

- **Wo wird dieses Objekt abgelegt?** Das ist eine **Kategorie**.
- **Welche endliche Liste versuche ich zu vervollständigen?** Das ist ein **Set**.
- **Zu welchem Franchise gehört das, über alles hinweg, was ich sammle?** Das ist eine **Reihe**.

## Kategorien: Ablage innerhalb einer Sammlung

Eine Kategorie legt Objekte innerhalb einer @doc(collections.overview, "Sammlung") ab, und Kategorien können sich verschachteln. Emmas Sammlung "Meine Comics" hat eine Marvel-Kategorie, mit Spider-Man und X-Men darin verschachtelt. Beim Durchsuchen einer Kategorie werden nur ihre Objekte angezeigt, mit eigenen Zählungen und Statistiken.

Eine Kategorie gehört zu ihrer Sammlung und kann nicht darüber hinausreichen. Das Löschen einer Kategorie löscht nie die Objekte darin; sie werden einfach unkategorisiert.

## Sets: eine endliche Liste zum Vervollständigen

Ein Set ist eine Checkliste mit einem Ende. Emma möchte die ersten zehn Ausgaben einer Serie, also erstellt sie ein Set mit einem **Zielwert** von zehn. Während sie Objekte zum Set hinzufügt, zeigt KolleK Besitz gegen Ziel, und die Vollständigkeit fließt in ihre @doc(insights.collectionStatistics, "Sammlungsstatistiken") ein.

Wie Kategorien gehört ein Set zu einer Sammlung. Nur ein Set mit einem Zielwert über null zählt zu den Vollständigkeitsstatistiken; ein Set ohne einen ist einfach eine benannte Gruppierung.

## Reihen: ein Franchise über Sammlungen hinweg

Eine Reihe verbindet alles aus einem Franchise, wo auch immer es lebt. Harry Potter erstreckt sich über Emmas Bücher, ihre Filme und ihr LEGO, drei verschiedene Sammlungen. Eine Reihe reicht über alle davon hinweg, und ihre Seite zeigt die verknüpften Objekte gruppiert nach der Sammlung, in der sie jeweils leben, zusammen damit, wie viele Sammlungen sie berühren.

Eine Reihe verfolgt keine Vollständigkeit. Sie beantwortet "zeig mir alles zu Harry Potter", nicht "bin ich fertig?".

## Nebeneinander

| | Kategorie | Set | Reihe |
|---|---|---|---|
| Umfang | Eine Sammlung | Eine Sammlung | Das ganze Konto |
| Verschachtelt | Ja | Nein | Nein |
| Verfolgt Vollständigkeit | Nein | Ja, mit Zielwert | Nein |
| Beantwortete Frage | Wo wird das abgelegt | Zu welcher Liste gehört das | Zu welchem Franchise gehört das |

Ein Objekt kann alle drei gleichzeitig nutzen: abgelegt unter Marvel, Teil des Sets "Erste zehn Ausgaben" und verknüpft mit der Reihe Spider-Man.

Willst du etwas noch Leichteres als das, übergreifende Beschriftungen ganz ohne Struktur, dafür gibt es @doc(tags.overview, "Tags").

## Wie geht es weiter

- Baue einen Ablagebaum in @doc(categories.organizeItems).
- Jage einer vollständigen Serie hinterher in @doc(sets.trackCompletion).
- Verbinde ein Franchise in @doc(series.groupFranchise).
