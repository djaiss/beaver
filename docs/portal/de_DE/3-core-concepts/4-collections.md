# Sammlungen

Eine Sammlung ist eine übergeordnete Gruppe, die du benennst, und das Erste, was du beim Start des Katalogisierens erstellst. Diese Seite erklärt, was eine Sammlung trägt, und die Entscheidungen, die du beim Erstellen einer treffen musst, damit du sie sicher aufsetzen kannst.

## Was eine Sammlung ist

Eine Sammlung enthält verwandte @doc(items.itemsVsCopies, "Objekte"). Du entscheidest, was "verwandt" bedeutet: Emma führt "Meine Comics", Noah führt "Schallplatten", Priya führt "Uhren" und eine zweite Sammlung namens "Geerbte Stücke". Ein Konto kann so viele Sammlungen enthalten, wie du möchtest.

Die meisten Sammler legen eine Sammlung pro Hobby an. Eine weitere Unterteilung (etwa eine Sammlung pro Verlag) funktioniert meist besser mit @doc(organizing.categoriesSetsAndSeries, "Kategorien") innerhalb einer einzelnen Sammlung.

## Was eine Sammlung trägt

Beim Erstellen oder Bearbeiten einer Sammlung wählst du:

- **Einen Namen**, wie "Weinkeller".
- **Eine Beschreibung**, optional, für Kontext.
- **Ein Emoji** aus einer kleinen, festen Palette (📚 für Bücher, 💿 für Musik, 🍷 für Wein und so weiter), das in Listen neben dem Namen angezeigt wird.
- **Eine Währung.** Jede Sammlung hat ihre eigene, gewählt aus der unterstützten Liste (USD, EUR, GBP, JPY und mehr). Sie kann von der Standardwährung des Kontos abweichen, was praktisch ist, wenn du Schallplatten in Euro, aber Wein in Dollar kaufst. Geld, das an Exemplaren dieser Sammlung erfasst wird, verwendet sie.
- **Eine Sichtbarkeit**: privat, mit dem Konto geteilt oder öffentlich per Link. Sichtbarkeit verdient eine eigene Erklärung, siehe @doc(sharing.overview).
- **Einen oder mehrere @doc(collectionTypes.overview, "Sammlungstypen").** Das entscheidet, welche benutzerdefinierten Felder die Objekte in dieser Sammlung nutzen können. Aktiviere den Comics-Typ für eine Comic-Sammlung, und jedes Objekt kann eine Ausgabennummer und einen Verlag erfassen.

Nur der Name ist erforderlich. Alles andere kann später angepasst werden.

## Was darin lebt

Eine Sammlung besitzt ihre Objekte, ihre @doc(organizing.categoriesSetsAndSeries, "Kategorien") und ihre @doc(organizing.categoriesSetsAndSeries, "Sets"). Sie merkt sich außerdem, pro Person, wie jedes Mitglied sie am liebsten ansieht (Raster, Liste oder Tabelle), sodass Emmas Tabellenansicht Sams Rasteransicht nie stört.

## Wie geht es weiter

- Verstehe, was in eine Sammlung hineingehört, in @doc(items.itemsVsCopies).
- Erstelle eine echte in @doc(collections.manage).
- Entscheide, wer sie sehen kann, in @doc(collections.share).
