# Tags

Tags sind der leichteste Weg, Objekte zu beschriften. Keine Hierarchie, keine Struktur, nur Worte, die du anheftest und später danach filterst. Diese Seite erklärt, wofür sie sich eignen und wie sie sich von den anderen Gruppierungswerkzeugen unterscheiden.

## Was ein Tag ist

Ein Tag ist eine kurze, frei formulierte Beschriftung, wie "Signiert", "Erstausgabe" oder "Zu versichern". Tags gelten für das ganze Konto, über alle Sammlungen hinweg: Erstelle "Signiert" einmal, und es steht sowohl bei Emmas Comics als auch bei Noahs Schallplatten zur Verfügung. Ein Objekt kann so viele Tags tragen, wie du möchtest.

Tags glänzen bei Eigenschaften, die sich über deinen gesamten Katalog erstrecken. "Signiert" trifft auf Comics, Bücher und Platten zu. Eine @doc(organizing.categoriesSetsAndSeries, "Kategorie") könnte das nie ausdrücken, weil Kategorien innerhalb einer Sammlung leben.

## Tags gegenüber den anderen Gruppierungen

- **Kategorien** legen ein Objekt an einem Ort innerhalb einer Sammlung ab und können sich verschachteln. Tags kennen keine Hierarchie und reichen überallhin.
- **Sets** verfolgen eine endliche Liste bis zur Vollständigkeit. Tags zählen nichts.
- **Reihen** verbinden ein Franchise. Tags tragen keine Bedeutung über das Wort selbst hinaus.

Eine gute Faustregel: Beantwortet die Beschriftung "wo wird das abgelegt", nutze eine Kategorie. Markiert sie eine Eigenschaft oder einen Zustand ("Signiert", "Duplikat", "Zu verkaufen"), nutze einen Tag.

## Woher Tags kommen

Jeder mit Zugang als Bearbeiter oder Eigentümer kann beim Bearbeiten eines Objekts spontan einen Tag erstellen oder aus der vorhandenen Liste wählen. Eigentümer und Bearbeiter können die geteilte Liste außerdem aus den Kontoeinstellungen pflegen, behandelt in @doc(tags.manageAccount).

## Einen Tag löschen

Das Löschen eines Tags entfernt die Beschriftung von jedem Objekt, das ihn trug. Die Objekte selbst bleiben unberührt.

:::note
Das Löschen eines Tags gilt kontoweit. Entferne "Signiert", und es verschwindet gleichzeitig von jedem Objekt in jeder Sammlung.
:::

## Wie geht es weiter

- Wende Tags an und filtere danach in @doc(items.tagAndFind).
- Halte die geteilte Liste übersichtlich in @doc(tags.manageAccount).
- Vergleiche mit den strukturierten Gruppierungen in @doc(organizing.categoriesSetsAndSeries).
