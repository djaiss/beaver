# Standorte

Ein Standort beantwortet die Frage, die sich jede wachsende Sammlung irgendwann stellt: "Wo habe ich das hingelegt?" Diese Seite erklärt, wie KolleK physische Aufbewahrung modelliert.

## Was ein Standort ist

Ein Standort ist ein Ort, an dem ein @doc(items.itemsVsCopies, "Exemplar") physisch lebt: ein Zimmer, ein Regal, eine Kiste, ein Safe. Jeder Standort kann ein Emoji tragen, damit er in Listen auf einen Blick erkennbar ist.

Standorte verschachteln sich so tief wie nötig, sodass sie deinen tatsächlichen Raum widerspiegeln können. Noah modelliert seinen so: Wohnzimmer, darin Regal A, darin Kiste 3. Fragt er sich, wo eine Platte ist, ist die Antwort so genau wie seine Karte.

Standorte gelten kontoweit. Definiere "Vitrine" einmal, und jede Sammlung kann dort Exemplare aufbewahren, was der Realität entspricht: Ein Regal kann Comics und Münzen nebeneinander tragen.

## Standorte hängen an Exemplaren, nicht an Objekten

Ein Objekt ist eine Idee, also ist es das Exemplar, das irgendwo sitzt. Emmas zwei Exemplare desselben Comics leben an verschiedenen Orten: eines in Comic-Kiste 1, eines gerahmt an der Wand. Jedes Exemplar zeigt auf seinen eigenen aktuellen Standort.

Ein frisches Konto kommt mit einigen Startstandorten (Wohnzimmer, Lager, Vitrine, Garage, Büro). Benenne sie um, verschachtele darunter, oder ersetze sie durch deine eigenen.

## Umzüge werden erinnert

Wenn du ein Exemplar umziehst, überschreibt KolleK nicht einfach den alten Standort. Es erfasst den Umzug, sodass das Exemplar eine Spur von jedem Ort behält, an dem es war, und wann. Der aktuelle Standort ist einfach der jüngste Eintrag in dieser Spur. Das ist Teil von @doc(copyHistory.concept, "der Historie eines Exemplars"), und die Anleitung dazu findest du in @doc(copies.move).

## Wie geht es weiter

- Baue deine Aufbewahrungskarte in @doc(locations.setup).
- Ziehe Dinge richtig um in @doc(copies.move).
- Sieh, wo Standorte im Formular für Exemplare auftauchen, in @doc(copies.track).
