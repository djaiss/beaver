# Einen inaktiven Benutzer automatisch löschen

KolleK bietet eine optionale Einstellung, die deinen Benutzer nach einer langen Phase der Inaktivität automatisch löscht, etwa sechs Monate. Sie existiert für Personen, die lieber keinen ruhenden Zugang auf einem Server zurücklassen möchten, zu dem sie vielleicht nie zurückkehren.

## Wie es funktioniert

- Die Einstellung ist **standardmäßig aus, bis du sie aktivierst**, und sie betrifft nur dich. Niemand kann sie in deinem Namen aktivieren, und deine Wahl wirkt sich auf kein anderes Mitglied des Kontos aus.
- Ist sie aktiviert, prüft eine tägliche Kontrolle, wann du zuletzt aktiv warst. Sind mehr als etwa sechs Monate vergangen, wird dein Benutzer gelöscht, genau wie wenn du @doc(users.deleteSelf, "ihn selbst gelöscht") hättest.
- Jede Aktivität setzt die Uhr zurück. Anmelden reicht schon aus.

## Ein- oder ausschalten

Finde die Option in den Sicherheitseinstellungen deines Profils, wähle Ja oder Nein und speichere. Du kannst es dir jederzeit anders überlegen, bevor die Löschung stattfindet.

:::warning
Löst die Inaktivitätslöschung aus, wird dein Benutzer endgültig entfernt, und dein Zugang funktioniert nicht mehr, ohne eine weitere Warn-E-Mail. Aktiviere das nur, wenn du wirklich willst, dass ein ruhender Zugang von selbst verschwindet.
:::

## Passt das zu dir

Aktiviere es, wenn du KolleK ausprobierst, oder datenschutzbewusst genug bist, dass dich ein verlassener Zugang stört. Lass es aus, wenn dieser Katalog ein langfristiges Archiv ist, das du vielleicht ein Jahr lang nicht anfasst und trotzdem noch auf dich warten soll. Sammlungen sind geduldige Dinge, eine Pause von acht Monaten zwischen Katalogisierungssitzungen ist in diesem Hobby normal.

## Wie es weitergeht

- Lösche deinen Benutzer stattdessen bewusst in @doc(users.deleteSelf).
- Sieh dir deine eigenen letzten Aktivitäten an in @doc(activity.logAndSentEmails, "Dein persönliches Aktivitätsprotokoll und gesendete E-Mails").
