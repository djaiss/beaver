# Objekte taggen und finden

@doc(tags.overview) sind der leichteste Weg, Objekte mit einer Gemeinsamkeit zu kennzeichnen, über jede Sammlung in deinem Konto hinweg. Diese Seite behandelt das Anwenden, das Entfernen und Gewohnheiten, die sie nützlich halten.

## Tags zu einem Objekt hinzufügen

Nutze im Objektformular das **Tags**-Feld:

- **Wähle ein bestehendes Tag** aus der gemeinsamen Liste deines Kontos, oder
- **Tippe ein neues ein**, es wird sofort erstellt, bereit zur Wiederverwendung überall.

Ein Objekt kann so viele Tags tragen, wie du möchtest. Emma taggt ihre signierten Hefte mit "Signiert", egal ob sie in ihrer Comics-Sammlung oder ihrer Graphic-Novels-Sammlung liegen. Das Tag ist in beiden dasselbe Label.

## Ein Tag von einem Objekt entfernen

Bearbeite das Objekt und entferne das Tag aus dem Tags-Feld. Das Tag selbst bleibt in der Kontoliste erhalten, und jedes andere Objekt, das es trägt, bleibt unberührt.

## Die gemeinsame Liste pflegen

Da Tags kontoweit geteilt werden, wird die Liste selbst an einem Ort verwaltet, in den Kontoeinstellungen. Das Umbenennen und Löschen von Tags dort wird in @doc(tags.manageAccount) behandelt.

:::note
Löschst du ein Tag aus dem Konto, wird dieses Label von jedem Objekt entfernt, das es trug. Die Objekte selbst sind ansonsten nicht betroffen.
:::

## Gewohnheiten, die Tags nützlich halten

- **Tagge Eigenschaften, keine Kategorien.** "Signiert", "Erstauflage", "Braucht Reparatur", "Zu verkaufen" sind gute Tags. "Comics" ist eine Sammlung, und "Marvel" ist wahrscheinlich eine @doc(categories.organizeItems, "Kategorie").
- **Nutze Bestehendes, bevor du Neues erfindest.** Zwei Schreibweisen derselben Idee ("Signiert" und "Handsigniert") teilen deinen Katalog in zwei Hälften. Wähle bestehende Tags, wenn sie passen.
- **Halte die Liste kurz.** Eine Handvoll Tags, nach denen du wirklich filterst, schlägt fünfzig, die du dir merken musst.
