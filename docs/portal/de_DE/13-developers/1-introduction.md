# Entwickler und die API

KolleK liefert eine vollständige JSON-API, die die Webanwendung eins zu eins spiegelt: Alles, was du in der App tun kannst, kannst du auch mit einem Token und einem HTTP-Client tun. Dieser Abschnitt bringt dich zur Authentifizierung und Orientierung. Die vollständige Endpunktreferenz wird aus dem Code generiert und ist immer aktuell, daher wiederholen diese Seiten sie bewusst nicht.

Lies die Seiten in dieser Reihenfolge:

1. @doc(api.overview). Was die API abdeckt, wie sie aufgebaut ist, und wo die generierte Referenz liegt.
2. @doc(api.authenticate). Hole dir ein Token und stelle deine erste Anfrage.
3. @doc(api.rateLimitsAndConventions). Paginierung, Geldbeträge, Drosselung und Fehlerverhalten.
4. @doc(webhooks.overview). Registriere heute schon Endpunkte und verstehe genau, was schon (und noch nicht) auslöst.

Für Details zu einzelnen Endpunkten (Parameter, Antwortformate, Beispiele) nutze die generierte Referenz unter `/docs/api` auf deiner Instanz.
