# Sammlungstypen und benutzerdefinierte Felder einrichten

Ein @doc(collectionTypes.overview, "Sammlungstyp") legt fest, welche Details ein Objekt erfassen kann. Ein Comic braucht eine Heftnummer und einen Verlag. Eine Schallplatte braucht einen Künstler und ein Pressjahr. Diese Seite zeigt dir, wie du einen Typ anlegst, ihm benutzerdefinierte Felder hinzufügst und lange Formulare mit Feldgruppen übersichtlich hältst.

Du brauchst die Rolle Bearbeiter oder Eigentümer, um Typen zu verwalten. Typen gelten für das ganze Konto, sodass ein Typ, den du einmal sorgfältig eingerichtet hast, von beliebig vielen Sammlungen wiederverwendet werden kann.

## Mit den fertigen Typen starten

Ein frisches Konto enthält bereits ein Dutzend fertiger Typen (Comics, Sammelkarten, Schallplatten, CD, DVD, Münzen, Briefmarken, Bücher, Actionfiguren / Spielzeug, Videospiele, Uhren und Wein), jeder mit sinnvoll gruppierten Feldern. Bevor du von Grund auf neu baust, öffne den Typ, der deinem Hobby am nächsten kommt, und passe ihn an. Ein Feld umzubenennen oder hinzuzufügen geht schneller, als bei null anzufangen.

## Einen Typ erstellen

Noah sammelt Schallplatten und möchte einen Typ für Konzertplakate, den die vorgegebenen Typen nicht abdecken.

::::steps
:::step title="Sammlungstypen öffnen"
Gehe zu deinen Kontoeinstellungen und öffne **Sammlungstypen**.

::screenshot{label="Liste der Sammlungstypen in den Kontoeinstellungen"}
:::

:::step title="Den Typ erstellen"
Wähle **Neuer Typ**, gib ihm einen Namen (Noah tippt "Konzertplakate") und wähle eine Farbe. Die Farbe hilft dir, Typen in Listen auf einen Blick zu unterscheiden.
:::

:::step title="Die ersten Felder hinzufügen"
Öffne den neuen Typ und füge für jedes Detail, das du erfassen möchtest, ein benutzerdefiniertes Feld hinzu. Wähle für jedes Feld einen Namen und einen Feldtyp.

::screenshot{label="Typ-Editor mit der Feldliste"}
:::
::::

Der Typ-Editor speichert automatisch. Es gibt keinen eigenen Speichern-Button, den du dir merken musst, jede Änderung wird sofort übernommen.

## Den richtigen Feldtyp wählen

Jedes benutzerdefinierte Feld hat einen von sechs Feldtypen:

- **Text** für frei formulierte Details, etwa einen Künstler oder einen Veranstaltungsort.
- **Zahl** für Mengen und Maße, etwa eine Heftnummer oder eine Auflage.
- **Datum** für alles Kalenderbezogene, etwa ein Konzertdatum.
- **Ja / Nein** für einfache Kennzeichnungen, etwa "Signiert" oder "Erstausgabe".
- **Auswahl** für eine feste Liste von Optionen, die du selbst definierst, etwa einen Verlag oder eine Note. Optionen halten die Daten konsistent, weil alle aus derselben Liste wählen, statt Varianten einzutippen.
- **Bewertung** für eine persönliche Wertung von einem bis fünf Sternen.

Bevorzuge **Auswahl** gegenüber **Text**, sobald die möglichen Werte eine bekannte Liste sind. "Marvel" und "marvel comics" sehen für dich gleich aus, für einen Filter aber nicht.

## Formulare mit Feldgruppen übersichtlich halten

Felder lassen sich in benannten Gruppen organisieren, und jede Gruppe erscheint als eigener Abschnitt im Objektformular. Der fertige Comics-Typ gruppiert seine Felder zum Beispiel in "Verlagsinfo" und "Zustand & Bewertung". Nicht gruppierte Felder erscheinen zuerst.

Erstelle eine Gruppe, gib ihr einen Namen und verschiebe Felder hinein. Du kannst sowohl die Felder innerhalb einer Gruppe als auch die Gruppen selbst neu anordnen, sodass das Formular in der Reihenfolge angezeigt wird, die für dein Hobby sinnvoll ist.

:::note
Gruppen wirken sich nur auf die Darstellung des Objektformulars aus. Sie ändern nichts an den Daten selbst, du kannst also jederzeit umsortieren.
:::

## Den Typ an Sammlungen anhängen

Ein Typ tut nichts, bis eine @doc(collections.overview, "Sammlung") ihn aktiviert. Wenn du eine Sammlung erstellst oder bearbeitest, wählst du, welche Typen gelten sollen. Eine Sammlung kann mehrere aktivieren, und derselbe Typ kann vielen Sammlungen dienen. Sobald ein Typ aktiviert ist, können Objekte in dieser Sammlung ihn wählen und seine Felder ausfüllen.

## Wie es weitergeht

- Teile ein Setup, auf das du stolz bist, oder übernimm eines mit @doc(collectionTypes.importExport).
- Setze die Felder in @doc(items.addAndEdit) ein.
- Runde dein Setup mit @doc(locations.setup) ab.
