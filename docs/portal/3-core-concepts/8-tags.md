# Tags

Tags are the lightest way to label items. No hierarchy, no structure, just words you attach and filter by later. This page explains what they are good at and how they differ from the other grouping tools.

## What a tag is

A tag is a short, free form label, such as "Signed", "First Issue", or "To insure". Tags are shared across every collection in the account: create "Signed" once, and it is available on Emma's comics and Noah's vinyl alike. An item can carry as many tags as you want.

Tags shine for qualities that cut across your whole catalogue. "Signed" applies to comics, books, and records. A @doc(organizing.categoriesSetsAndSeries, "category") could never express that, because categories live inside one collection.

## Tags versus the other groupings

- **Categories** file an item in one place within a collection, and can nest. Tags impose no hierarchy and reach everywhere.
- **Sets** track a finite list to completion. Tags count nothing.
- **Series** tie a franchise together. Tags carry no meaning beyond the word itself.

A good rule: if the label answers "where does this file", use a category. If it marks a quality or a state ("Signed", "Duplicate", "For sale"), use a tag.

## Where tags come from

Anyone with editor or owner access can create a tag on the spot while editing an item, or pick from the existing list. Owners and editors can also curate the shared list from account settings, covered in @doc(tags.manageAccount).

## Deleting a tag

Deleting a tag removes the label from every item that carried it. The items themselves are untouched.

:::note
Tag deletion is account wide. Remove "Signed" and it disappears from every item in every collection at once.
:::
