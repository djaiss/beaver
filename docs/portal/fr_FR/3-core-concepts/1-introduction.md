# Concepts fondamentaux

KolleK repose sur un modèle de données précis, et il mérite vingt minutes de votre temps. Une fois que vous comprenez la poignée d'idées présentées dans cette section, chaque page de tâches du reste de la documentation devient évidente. Lisez ces pages une fois, tôt, et revenez-y dès qu'un terme vous semble flou.

Lisez les pages dans cet ordre.

1. @doc(kollek.howOrganized). Tout le modèle mental sur une seule page.
2. @doc(accounts.usersAndRoles). L'espace de travail, les personnes qui s'y trouvent, et ce que chacune peut y faire.
3. @doc(collections.overview). Les groupes nommés qui contiennent tout ce que vous cataloguez.
4. @doc(items.itemsVsCopies). L'idée la plus importante de KolleK.
5. @doc(collectionTypes.overview). Comment vous décidez ce que chaque type d'objet enregistre.
6. @doc(organizing.categoriesSetsAndSeries). Trois outils de regroupement, chacun répondant à une question différente.
7. @doc(tags.overview). La façon la plus légère d'étiqueter et de trier les objets.
8. @doc(locations.overview). Comment le stockage physique est modélisé.
9. @doc(conditions.overview). Comment l'état d'un exemplaire est évalué.
10. @doc(copyHistory.concept). Où vivent la valeur, les mouvements et la provenance.
11. @doc(sharing.overview). Qui peut voir une collection, et quand.
12. @doc(dataSafety.howProtected). Le chiffrement, la piste d'audit, et leurs limites.

Si vous n'avez que cinq minutes, lisez @doc(items.itemsVsCopies). C'est la page sur laquelle repose constamment le reste de la documentation.
