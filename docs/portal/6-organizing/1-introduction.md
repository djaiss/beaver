# Organizing Your Catalogue

Adding items is only half the work. This section covers the structure around them: the types and custom fields that decide what each item records, the categories, sets, and series that group them, the locations that map your physical storage, the tags that cut across everything, and the photo library that keeps your images tidy. Most of these are set up once, then refined as your collection grows.

Read the pages in whatever order matches what you are setting up:

1. [Set up collection types and custom fields](2-set-up-collection-types-and-custom-fields.md). Define what each kind of item records.
2. [Import and export a collection type](3-import-and-export-a-collection-type.md). Move a type definition in or out as JSON.
3. [Organize items with categories](4-organize-items-with-categories.md). Build a nested filing tree inside a collection.
4. [Track a set to completion](5-track-a-set-to-completion.md). Know what you own versus what you still need.
5. [Group a franchise with series](6-group-a-franchise-with-series.md). Tie items together across collections.
6. [Set up your locations](7-set-up-your-locations.md). Map where things physically live.
7. [Manage account tags](8-manage-account-tags.md). Curate the shared tag vocabulary.
8. [Browse and manage your photo library](9-photo-library.md). Work with every image in one place.

If the difference between categories, sets, and series is fuzzy, start with the concept page on [categories, sets, and series](../3-core-concepts/7-categories-sets-and-series.md) before the how to guides.
